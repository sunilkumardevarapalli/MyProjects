Handling **asynchronous operations** in JavaScript is one of the most important topics for experienced developers. JavaScript is single-threaded, so long-running tasks (API calls, file reads, timers, database calls) are handled asynchronously without blocking the main thread.

## 1. Callback Functions (Old Approach)

A callback is a function passed to another function and executed later.

```javascript
function fetchData(callback) {
    setTimeout(() => {
        callback("Data received");
    }, 2000);
}

fetchData((result) => {
    console.log(result);
});

console.log("Program continues...");
```

**Output**

```
Program continues...
Data received
```

### Problems

* Callback hell
* Difficult to maintain
* Error handling becomes complex

---

## 2. Promises

Promises represent a value that may be available now, later, or never.

### Creating a Promise

```javascript
function fetchUser() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve("User data");
        }, 2000);
    });
}

fetchUser()
    .then(data => console.log(data))
    .catch(err => console.log(err));
```

### Promise States

1. Pending
2. Fulfilled (Resolved)
3. Rejected

---

## 3. Promise Chaining

```javascript
getUser()
    .then(user => getOrders(user.id))
    .then(orders => getPayment(orders))
    .then(payment => console.log(payment))
    .catch(err => console.log(err));
```

This avoids callback nesting.

---

## 4. Async/Await (Modern Approach)

```javascript
function fetchData() {
    return new Promise(resolve => {
        setTimeout(() => {
            resolve("Data received");
        }, 2000);
    });
}

async function process() {
    try {
        const result = await fetchData();
        console.log(result);
    } catch (err) {
        console.log(err);
    }
}

process();
```

### Advantages

* Cleaner syntax
* Easier error handling
* Looks synchronous

---

## 5. Parallel Execution with Promise.all()

Without `Promise.all()`:

```javascript
const user = await getUser();
const orders = await getOrders();
```

Runs sequentially.

With `Promise.all()`:

```javascript
const [user, orders] = await Promise.all([
    getUser(),
    getOrders()
]);

console.log(user, orders);
```

Both execute concurrently.

---

## 6. Promise.allSettled()

Executes all promises and returns results regardless of failures.

```javascript
const results = await Promise.allSettled([
    Promise.resolve("Success"),
    Promise.reject("Failed")
]);

console.log(results);
```

Output:

```javascript
[
 {status: "fulfilled", value: "Success"},
 {status: "rejected", reason: "Failed"}
]
```

Useful when one failure shouldn't stop others.

---

## 7. Promise.race()

Returns the first settled promise.

```javascript
Promise.race([
    fetchFromServer1(),
    fetchFromServer2()
]).then(result => console.log(result));
```

Useful for timeout scenarios.

## 8. Error Handling

```javascript
async function getData() {
    try {
        const response = await fetch(url);

        if (!response.ok) {
            throw new Error("API failed");
        }

        const data = await response.json();
        return data;

    } catch (error) {
        console.error(error.message);
    }
}
```

## 9. Real-Time API Call Example

```javascript
async function fetchUsers() {
    try {
        const response = await fetch(
            "https://jsonplaceholder.typicode.com/users"
        );

        const users = await response.json();

        console.log(users);

    } catch (err) {
        console.log("Error:", err.message);
    }
}

fetchUsers();
```

---

## 10. Handling Multiple API Calls Efficiently

```javascript
async function loadDashboard() {
    try {
        const [users, posts, comments] = await Promise.all([
            fetch('/users').then(r => r.json()),
            fetch('/posts').then(r => r.json()),
            fetch('/comments').then(r => r.json())
        ]);

        console.log(users, posts, comments);

    } catch (err) {
        console.error(err);
    }
}
```

## Event Loop Execution Example

```javascript
console.log("Start");

setTimeout(() => {
    console.log("Timeout");
}, 0);

Promise.resolve().then(() => {
    console.log("Promise");
});

console.log("End");
```

### Output

```
Start
End
Promise
Timeout
```

Because:

1. Synchronous code executes first.
2. Microtask Queue (Promises) executes next.
3. Macrotask Queue (`setTimeout`) executes last.

# For 10–15 Years Experience Interviews

**Preferred approach:** **Async/Await + Promise.all + try/catch**

```javascript
async function processRequests() {
    try {
        const [users, products, orders] = await Promise.all([
            fetchUsers(),
            fetchProducts(),
            fetchOrders()
        ]);

        return {
            users,
            products,
            orders
        };

    } catch (error) {
        console.error("Request failed:", error);
    }
}
```

### Interview Answer

> "JavaScript asynchronous operations can be handled using callbacks, promises, and async/await. In modern applications, I prefer async/await with try-catch for readability and Promise.all for parallel execution. I also leverage Promise.allSettled and Promise.race depending on error handling and timeout requirements. Understanding the Event Loop, Microtask Queue, and Macrotask Queue is essential for writing efficient asynchronous code."
