In JavaScript, the **Event Lifecycle** describes what happens from the moment an event occurs (click, keypress, etc.) until the event reaches the target element and finishes processing.

## 1. Event Capturing Phase (Trickling Down)

* The event starts from the `window` object and travels down through parent elements to the target element.
* Also called **capturing phase**.


<div id="parent">
    <button id="child">Click Me</button>
</div>


script
document.getElementById("parent")
    .addEventListener("click", () => {
        console.log("Parent Capturing");
    }, true); // true = capturing

document.getElementById("child")
    .addEventListener("click", () => {
        console.log("Child Capturing");
    }, true);


When button is clicked:


Parent Capturing
Child Capturing


---

## 2. Target Phase

* The event reaches the actual element that triggered it.
* Event handlers registered on the target execute.

script
document.getElementById("child")
    .addEventListener("click", () => {
        console.log("Button Clicked");
    });


Output:


Button Clicked


---

## 3. Bubbling Phase

* After reaching the target, the event bubbles up from child to parent elements.
* This is the default behavior.

script
document.getElementById("parent")
    .addEventListener("click", () => {
        console.log("Parent Bubbling");
    });

document.getElementById("child")
    .addEventListener("click", () => {
        console.log("Child Bubbling");
    });


Output:


Child Bubbling
Parent Bubbling


---

## Example with Both Capturing and Bubbling


<div id="parent">
    <button id="child">Click Me</button>
</div>


script
parent.addEventListener("click", () => {
    console.log("Parent Capture");
}, true);

child.addEventListener("click", () => {
    console.log("Child Target");
});

parent.addEventListener("click", () => {
    console.log("Parent Bubble");
});


Output when button is clicked:


Parent Capture
Child Target
Parent Bubble

## Controlling Event Propagation

### `event.stopPropagation()`

Stops the event from continuing to parent elements.

script
child.addEventListener("click", (event) => {
    console.log("Button clicked");
    event.stopPropagation();
});


Output:


Button clicked


Parent handler will not execute.

---

### `event.preventDefault()`

Prevents the browser's default action.


<a href="https://google.com" id="link">Google</a>


script
link.addEventListener("click", (event) => {
    event.preventDefault();
    console.log("Navigation prevented");
});


---

## Interview Answer (Short)

**JavaScript Event Lifecycle consists of three phases:**

1. **Capturing Phase** – Event travels from `window` down to the target element.
2. **Target Phase** – Event reaches the element that triggered it.
3. **Bubbling Phase** – Event propagates back from the target element to its ancestors.

Methods like **`stopPropagation()`** and **`preventDefault()`** can control event behavior.
