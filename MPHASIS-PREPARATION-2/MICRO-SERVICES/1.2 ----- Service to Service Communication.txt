Step 2 GraphQL Controller (Resolver)

	This is the class that GraphQL invokes automatically.

		@Controller
		public class CustomerGraphQLController {

			@Autowired
			private CustomerService customerService;

			@QueryMapping
			public CustomerResponse customer(
					@Argument Integer id) {

				return customerService.getCustomer(id);

			}

		}
This annotation tells Spring GraphQL

"Whenever someone executes the GraphQL query named customer, call this method."

So this method is automatically invoked.


Step 3 Service

Now the service is called.

@Service
public class CustomerService {

    @Autowired
    private RestTemplate restTemplate;

    public CustomerResponse getCustomer(Integer id){

        User user =
                restTemplate.getForObject(
                        "http://USER-SERVICE/users/" + id,
                        User.class);

        Order[] orders =
                restTemplate.getForObject(
                        "http://ORDER-SERVICE/orders/customer/" + id,
                        Order[].class);

        return new CustomerResponse(
                user,
                Arrays.asList(orders));
    }
}

This service calls two REST APIs.

GET /users/1

GET /orders/customer/1

How does Spring know to call this method?

Suppose the GraphQL schema is:

type Query {

    customer(id:Int): CustomerResponse

}

Spring matches

customer

with

@QueryMapping
public CustomerResponse customer(...)

The method name (customer) matches the GraphQL query name (customer).

So when the client executes:

query{

    customer(id:1)

}

Spring internally performs something like:

customerGraphQLController.customer(1);

Then your service executes:

customerService.getCustomer(1);

Interview Question

Interviewer: Who invokes the CustomerService.getCustomer() method?

Answer:

The client never calls the service directly. The client sends a GraphQL query to the /graphql endpoint. Spring GraphQL processes the query, matches it with the corresponding @QueryMapping resolver method, invokes that resolver, and the resolver calls the CustomerService. The service then fetches data from multiple REST microservices using RestTemplate or WebClient, aggregates the data, and returns it. Finally, Spring GraphQL serializes the returned object into the GraphQL response.

This resolver layer (@QueryMapping, @MutationMapping, @SchemaMapping) is the GraphQL equivalent of a @RestController in a REST application.
