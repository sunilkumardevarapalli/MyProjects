

---------------------------------DB + Messaging Atomicity -------------------------------

	In SAGA Design Patteren: 
	
		Both DB update for Order and sending OrderCreatedEvent in one transaction. 
		
	@Transactional normally manages the database transaction.
		
	Dual Write Problem:

		DB committed, but message was not published
		or
		message published, but DB transaction rolled back
	
Best Solution: Transactional Outbox Pattern: with Scheduler and Debezium

	SAME DB TRANSACTION: ORDER TABLE,OUTBOX TABLE(Event) --> both save and commit in one Data base TRANSACTION
	
	Spring Boot scheduler reads the outbox table, Takes New Events and process it.
	
1. OUT BOX + Scheduler + Idempotent Consumer

2. Debezium: Insted of scheduler Debezium  read the log and send the event.

     +---- ORDER TABLE
      |
      +---- OUTBOX TABLE
               |
               v
          DB Transaction Log
               |
               v
            Debezium
               |
               v
             Kafka
			 
------------------------------------JPA/Hibernate Custom Queries

Custom Queries are used to when repository-derived methods are not sufficient 

for complex data retrieval, updates, joins, aggregations, or database-specific operations.

1. JPQL (Java Persistence Query Language)

	JPQL works with entity objects and their fields, not directly with database tables.
	
	@Query
		1. Queries with Entity and their Fields
		
		2. Native SQL Queries: mention nativeQuery = true
		
		3. Named Queries:
		
		4. Update and Delete Queries:@Modifying and @Transactional
		
		5. Projection Queries
		
	Dynamic Queries with Criteria API
	
	Pagination with Custom Query:
		
		@Query("SELECT e FROM Employee e WHERE e.salary > :salary")
		Page<Employee> findEmployees(@Param("salary") double salary,Pageable pageable);
		
		Page<Employee> page =repository.findEmployees(50000,PageRequest.of(0, 10));

----------------------------------------------Database Design – Pagination


Pagination is the technique of retrieving large datasets in smaller chunks (pages) instead of loading all records at once.


	1. Offset Pagination
		✅ Simple
		❌ Slow for large datasets

	2.Keyset Pagination
		✅ Fast
		✅ Scalable
		❌ No random page jump

	3.Cursor Pagination
		✅ Best for APIs and infinite scrolling
		✅ Consistent results
		✅ High performance
