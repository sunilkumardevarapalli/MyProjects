# What is Carrier Thread?

A **Carrier Thread is a Platform Thread used by the JVM to execute Virtual Threads**.

Example:

```text
Virtual Thread VT-1
        |
        | mounted on
        v
Carrier Thread
        |
        | backed by
        v
OS Thread
        |
        v
CPU
```

The **Virtual Thread never directly runs on the CPU**.

It runs on a **Carrier Thread**.

The Carrier/Platform Thread runs on an **OS Thread**.

---

# Carrier Thread vs OS Thread

| Carrier Thread                      | OS Thread                |
| ----------------------------------- | ------------------------ |
| JVM/Java concept                    | Operating System concept |
| Platform Thread acting as a carrier | Native kernel thread     |
| Executes Virtual Threads            | Scheduled by OS          |
| Managed/used by JVM                 | Managed by OS            |
| Backed by OS Thread                 | Executes on CPU          |

So:

```text
Carrier Thread
      |
      | mapped to / backed by
      v
OS Thread
```

### In simple interview language

> **A carrier thread is a Java platform thread used by the JVM to execute virtual threads. A platform thread is backed by an operating-system thread.**

# Example with 5 Virtual Threads

Suppose:

```text
5 Virtual Threads

VT-1
VT-2
VT-3
VT-4
VT-5
```

And:

```text
2 Carrier Threads
```

Initially:

```text
Carrier-1  ---> VT-1
Carrier-2  ---> VT-2
```

Mapping:

```text
VT-1 ---> Carrier-1 ---> OS Thread-1 ---> CPU
VT-2 ---> Carrier-2 ---> OS Thread-2 ---> CPU
```

Now VT-1 performs a blocking DB call:

```java
jdbcTemplate.query(...);
```

Conceptually:

```text
VT-1
 |
 v
Waiting for DB
```

The JVM can **unmount VT-1 from Carrier-1**:

```text
VT-1 ---> Waiting

Carrier-1 ---> FREE
```

Now VT-3 can run:

```text
VT-3
 |
 v
Carrier-1
 |
 v
OS Thread-1
 |
 v
CPU
```

So:

```text
Before

Carrier-1 ---> VT-1
Carrier-2 ---> VT-2


VT-1 blocked

Carrier-1 ---> VT-3
Carrier-2 ---> VT-2
```

Later, when the DB response comes:

```text
VT-1 becomes runnable
        |
        v
Available Carrier Thread
```

It does **not need to return to the same Carrier Thread**.

For example:

```text
Earlier:

VT-1 ---> Carrier-1

Later:

VT-1 ---> Carrier-2
```

# Important correction to your understanding

Don't say:

> **Carrier Thread and OS Thread are exactly the same.**

Instead say:

> **A Carrier Thread is a Java Platform Thread backed by an OS Thread.**

```text
Virtual Thread
      |
      | mounted on
      v
Carrier / Platform Thread
      |
      | backed by
      v
OS Thread
      |
      v
CPU
```

## Easy memory trick

```text
Virtual Thread  = Task
Carrier Thread  = Vehicle
OS Thread       = Road-level execution resource
CPU             = Engine executing instructions
```

**Final answer: Carrier Thread is not literally the OS Thread, but a carrier is a Java Platform Thread backed by an OS Thread.**
