## ✅ Entity: OutboxEvent

@Table(name = "outbox_event") // maps to outbox table
public class OutboxEvent {

    @Id
    private String id; // unique event id (UUID)

    private String aggregateId;   // orderId
    private String aggregateType; // ORDER, PAYMENT
    private String eventType;     // ORDER_CREATED

    @Lob // large object (JSON payload)

}

---

## ✅ Service (CRITICAL PART)

```java
@Service
public class OrderService {

    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private OutboxRepository outboxRepository;

    @Transactional // 🔥 ensures atomic DB transaction
    public void createOrder() {

        // 1️⃣ Create order object
        Order order = new Order();
        order.setStatus("CREATED");

        // 2️⃣ Save order in DB
        orderRepository.save(order);

        // 3️⃣ Create outbox event
        OutboxEvent event = new OutboxEvent();
        event.setId(UUID.randomUUID().toString()); // unique event id
        event.setAggregateId(order.getId().toString()); // link to order
        event.setAggregateType("ORDER"); // used by Debezium routing
        event.setEventType("ORDER_CREATED");

        // 4️⃣ Convert payload to JSON
        event.setPayload("{\"orderId\":\"" + order.getId() + "\"}");

        // 5️⃣ Save event in SAME transaction
        outboxRepository.save(event);
    }
}
```

---

## 🔥 What happens internally

```text
DB Transaction:
   INSERT order
   INSERT outbox_event
COMMIT
```

👉 After commit → DB writes to WAL → Debezium picks it

## ✅ Payment Service (CRITICAL LOGIC)

```java
@Service
public class PaymentService {

    @Autowired
    private ProcessedEventRepository processedRepo;

    @Autowired
    private OutboxRepository outboxRepo;

    @Transactional // 🔥 ensures idempotency + business logic atomic
    public void process(OrderEvent event) {

        // 1️⃣ IDEMPOTENCY CHECK
        if (processedRepo.existsById(event.getEventId())) {
            return; // duplicate → ignore
        }

        try {
            // 2️⃣ BUSINESS LOGIC
            System.out.println("Payment processed for order: " + event.getOrderId());

            // 3️⃣ SAVE SUCCESS EVENT
            OutboxEvent successEvent = new OutboxEvent();
            successEvent.setId(UUID.randomUUID().toString());
            successEvent.setAggregateId(event.getOrderId());
            successEvent.setAggregateType("PAYMENT");
            successEvent.setEventType("PAYMENT_SUCCESS");
            successEvent.setPayload(toJson(event));

            outboxRepo.save(successEvent);

        } catch (Exception ex) {

            // 4️⃣ SAVE FAILURE EVENT
            OutboxEvent failEvent = new OutboxEvent();
            failEvent.setId(UUID.randomUUID().toString());
            failEvent.setAggregateType("PAYMENT");
            failEvent.setEventType("PAYMENT_FAILED");

            outboxRepo.save(failEvent);

            throw ex; // 🔥 triggers Kafka retry
        }

        // 5️⃣ MARK AS PROCESSED
        processedRepo.save(new ProcessedEvent(event.getEventId()));
    }
}
```

## ✅ Service

```java
@Transactional
public void reserve(PaymentEvent event) {

    if (processedRepo.existsById(event.getEventId())) return;

    // business logic
    System.out.println("Stock reserved");

    // publish next event
    outboxRepo.save(new OutboxEvent(
        UUID.randomUUID().toString(),
        event.getOrderId(),
        "INVENTORY",
        "STOCK_RESERVED",
        toJson(event)
    ));

    processedRepo.save(new ProcessedEvent(event.getEventId()));
}
```

    return new DefaultErrorHandler(

# 🔥 6. FULL FLOW (Line-by-Line Thinking)

```text
1. OrderService.createOrder()
   → saves order + outbox (TX commit)

2. DB writes WAL

3. Debezium reads WAL
   → converts row → Kafka event

4. Kafka publishes ORDER_CREATED

5. PaymentService.consume()
   → checks idempotency
   → processes payment
   → writes PAYMENT_SUCCESS outbox

6. Debezium again
   → publishes PAYMENT_SUCCESS

7. Inventory consumes
   → reserves stock
```
