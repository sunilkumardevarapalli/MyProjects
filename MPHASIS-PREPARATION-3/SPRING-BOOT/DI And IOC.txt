			emailService.send();

With IoC

Spring creates and manages both beans:

@Service
public class EmailService {

    public void send() {
        System.out.println("Email sent");
    }
}

@Service
public class EmployeeService {

    @Autowired
    private EmailService emailService;

    public void sendWelcomeEmail() {
        emailService.send();
    }
}

Spring internally does something like:

EmailService emailService = new EmailService();

EmployeeService employeeService = new EmployeeService();

employeeService.emailService = emailService;  //wiring

Thus, Spring controls object creation and wiring. This is called Inversion of Control (IoC).

Dependency Injection (DI)

Dependency Injection is a way to achieve IoC.

Dependency Injection means Spring injects dependent objects into a bean instead of the bean creating them itself.
