 Spring allows us to specify cleanup methods externally using `@Bean(destroyMethod = "...")`.

@Bean(destroyMethod = "close")

public class PaymentSDK {

    public void initialize() {
        System.out.println("Payment SDK initialized");
    }

    public void disconnect() {
        System.out.println("Payment SDK disconnected");
    }
}


Spring Configuration:


@Configuration
public class AppConfig {

    @Bean(initMethod = "initialize",
          destroyMethod = "disconnect")
    public PaymentSDK paymentSDK() {

        return new PaymentSDK();
    }
}

# Summary

| Scenario               | Recommended                |
| ---------------------- | -------------------------- |
| Custom JDBC Connection | `@PreDestroy`              |
| Redis Connection       | `@PreDestroy`              |
| Mongo Client           | `@PreDestroy`              |
| Thread Pool            | `@PreDestroy`              |
| Framework Components   | `DisposableBean.destroy()` |
| Spring Infrastructure  | `DisposableBean.destroy()` |
| Third-party SDKs       | `destroyMethod`            |
| FTP Client             | `destroyMethod`            |
| Redis Client (Lettuce) | `destroyMethod`            |
| JMS Client             | `destroyMethod`            |
