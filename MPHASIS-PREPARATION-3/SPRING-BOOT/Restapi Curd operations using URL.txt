In Spring Boot, REST API CRUD operations are typically mapped to HTTP methods and URLs as follows:

| Operation | HTTP Method | URL               | Description           |
| --------- | ----------- | ----------------- | --------------------- |
| Create    | POST        | `/employees`      | Create a new employee |
| Read All  | GET         | `/employees`      | Get all employees     |
| Read One  | GET         | `/employees/{id}` | Get employee by ID    |
| Update    | PUT         | `/employees/{id}` | Update employee       |
| Delete    | DELETE      | `/employees/{id}` | Delete employee       |

### Example

#### Entity

```java
@Entity
public class Employee {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private double salary;

    // Getters and Setters
}
```

#### Repository

```java
@Repository
public interface EmployeeRepository extends JpaRepository<Employee, Long> {
}
```

#### Controller

```java
@RestController
@RequestMapping("/employees")
public class EmployeeController {

    @Autowired
    private EmployeeRepository repository;

    // Create
    @PostMapping
    public Employee createEmployee(@RequestBody Employee employee) {
        return repository.save(employee);
    }

    // Read All
    @GetMapping
    public List<Employee> getAllEmployees() {
        return repository.findAll();
    }

    // Read By Id
    @GetMapping("/{id}")
    public Employee getEmployee(@PathVariable Long id) {
        return repository.findById(id)
                .orElseThrow(() -> new RuntimeException("Employee not found"));
    }

    // Update
    @PutMapping("/{id}")
    public Employee updateEmployee(@PathVariable Long id,
                                   @RequestBody Employee employee) {

        Employee existing = repository.findById(id)
                .orElseThrow(() -> new RuntimeException("Employee not found"));

        existing.setName(employee.getName());
        existing.setSalary(employee.getSalary());

        return repository.save(existing);
    }

    // Delete
    @DeleteMapping("/{id}")
    public String deleteEmployee(@PathVariable Long id) {
        repository.deleteById(id);
        return "Employee deleted successfully";
    }
}
```

### Sample URLs

#### Create Employee

```http
POST /employees
```

Request Body:

```json
{
  "name": "Sunil",
  "salary": 50000
}
```

#### Get All Employees

```http
GET /employees
```

#### Get Employee By ID

```http
GET /employees/1
```

#### Update Employee

```http
PUT /employees/1
```

Request Body:

```json
{
  "name": "Sunil Kumar",
  "salary": 60000
}
```

#### Delete Employee

```http
DELETE /employees/1
```

### Interview Tip

Remember the mapping:

```java
@PostMapping      // CREATE
@GetMapping       // READ
@PutMapping       // UPDATE
@DeleteMapping    // DELETE
```

and URL pattern:

```text
GET     /resources
GET     /resources/{id}
POST    /resources
PUT     /resources/{id}
DELETE  /resources/{id}
```

This is the standard RESTful CRUD design expected in Spring Boot interviews.
