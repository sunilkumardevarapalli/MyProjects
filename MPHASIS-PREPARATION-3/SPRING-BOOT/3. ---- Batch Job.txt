What is Batch Processing?

	Batch processing executes a collection of jobs automatically without user interaction. Typical use cases include:

		Processing large CSV/Excel files
		Data migration
		ETL (Extract, Transform, Load)
		Report generation
		Database cleanup
		Scheduled financial transactions

Spring Batch Architecture: A Spring Batch job consists of:

		Job
		 └── Step
			  ├── ItemReader
			  ├── ItemProcessor
			  └── ItemWriter

Dependency: pring-boot-starter-batch

1. Job:Represents the entire batch process.
2. Step:A phase within a job.
3. ItemReader:Reads data from a source.
4. ItemProcessor:Transforms or validates data.
5. ItemWriter:Writes data to the destination.

Chunk Processing:A key feature of Spring Batch:

Postman sends a CSV file to a REST controller. 

	The controller stores the file and invokes JobLauncher.run(job, jobParameters). 

	Spring Batch starts the Job, which executes a Step. 

		The Step uses an ItemReader to read records, 
		
		an ItemProcessor to apply business logic, 

		and an ItemWriter to persist records. 

Based on the configured chunk size (e.g., 2), records are processed and committed in batches until the entire file is completed.

-------------------------------------------------------------------------------------------------------------------------------
Who calls Reader, Processor, Writer, and Step?

Spring Batch framework calls them internally. 
When jobLauncher.run() is executed, Spring creates a JobExecution, executes the configured Step, 
and the Step automatically invokes ItemReader, ItemProcessor, and ItemWriter according to the configured chunk size.

Spring internally performs something similar to:

jobLauncher.run(employeeJob, jobParameters);


createJobExecution();

for(each step in job){
    step.execute();
}

Now Step executes: below proccess repete until file records will complete. based on chunk.

Let's take a **real-world scenario** where a user uploads a file through **Postman**, and Spring Batch processes it.

```text

   v

   v

   |
   | Start Job
   v
Spring Batch Job
   |
   v

   |
   +--> Reader
   |
   +--> Processor
   |
   +--> Writer
   |
   v
Database
```

***

# Step 1: User Uploads File via Postman

Postman Request:

```http
POST /employees/upload
Content-Type: multipart/form-data
```

Body:

```text
file = employees.csv
```

CSV:

```csv
id,name,salary
1,John,5000
2,Alice,6000
3,Bob,7000
4,David,8000
5,Emma,9000
```

***

# Step 2: Controller Receives Request

```java
@RestController
@RequestMapping("/employees")
public class EmployeeController {

    @Autowired
    private JobLauncher jobLauncher;

    @Autowired
    private Job employeeJob;

    @PostMapping("/upload")
    public String uploadFile(
            @RequestParam("file") MultipartFile file)
            throws Exception {

        String filePath =
                "C:/uploads/" + file.getOriginalFilename();

        file.transferTo(new File(filePath));

        JobParameters jobParameters =
                new JobParametersBuilder()
                        .addString("filePath", filePath)
                        .addLong("time", System.currentTimeMillis())
                        .toJobParameters();

        jobLauncher.run(employeeJob, jobParameters);

        return "Batch Job Started";
    }
}
```

### What happens here?

1. Postman sends CSV.
2. Controller receives file.
3. File saved to local folder.
4. Controller invokes:

```java
jobLauncher.run(employeeJob, jobParameters);
```

This is the point where Spring Batch starts.

***

# Step 3: Job Starts

```java
@Bean
public Job employeeJob(JobRepository jobRepository,
                       Step employeeStep) {

    return new JobBuilder("employeeJob", jobRepository)
            .start(employeeStep)
            .build();
}
```

Execution:

```text
Controller
   |
jobLauncher.run(...)
   |
   v
Job Started
```

***

# Step 4: Reader Gets File Path

Reader reads the file uploaded by Postman.

```java
@Bean
@StepScope
public FlatFileItemReader<Employee> reader(
        @Value("#{jobParameters['filePath']}") String filePath) {

    FlatFileItemReader<Employee> reader =
            new FlatFileItemReader<>();

    reader.setResource(new FileSystemResource(filePath));

    return reader;
}
```

Job parameter:

```text
C:/uploads/employees.csv
```

Reader opens file.

***

# Step 5: Step Starts

```java
@Bean
public Step employeeStep(
        JobRepository jobRepository,
        PlatformTransactionManager transactionManager) {

    return new StepBuilder("employeeStep", jobRepository)
            .<Employee, Employee>chunk(2, transactionManager)
            .reader(reader(null))
            .processor(processor())
            .writer(writer())
            .build();
}
```

Chunk Size:

```java
.chunk(2)
```

Meaning:

```text
Read 2
Process 2
Write 2
Commit
```

***

# Step 6: Reader Reads First Chunk

File:

```text
1,John,5000
2,Alice,6000
3,Bob,7000
4,David,8000
5,Emma,9000
```

Reader executes internally:

```java
reader.read()
```

Returns:

```java
Employee(1,John,5000)
Employee(2,Alice,6000)
```

Chunk Full:

```text
[John, Alice]
```

***

# Step 7: Processor Executes

```java
@Component
public class EmployeeProcessor
        implements ItemProcessor<Employee, Employee> {

    @Override
    public Employee process(Employee employee) {

        employee.setSalary(
                employee.getSalary() + 1000);

        return employee;
    }
}
```

Execution:

```text
John 5000 -> 6000
Alice 6000 -> 7000
```

***

# Step 8: Writer Executes

```java
@Bean
public JdbcBatchItemWriter<Employee> writer(
        DataSource dataSource) {

    return new JdbcBatchItemWriterBuilder<Employee>()
            .dataSource(dataSource)
            .sql("insert into employee(id,name,salary) " +
                 "values(:id,:name,:salary)")
            .beanMapped()
            .build();
}
```

SQL executed:

```sql
insert into employee values(1,'John',6000)

insert into employee values(2,'Alice',7000)
```

## Interview Answer

If asked *"How does batch processing start from Postman?"*:

> Postman sends a CSV file to a REST controller. The controller stores the file and invokes `JobLauncher.run(job, jobParameters)`. Spring Batch starts the Job, which executes a Step. The Step uses an ItemReader to read records, an ItemProcessor to apply business logic, and an ItemWriter to persist records. Based on the configured chunk size (e.g., 2), records are processed and committed in batches until the entire file is completed.
