In Java, **Thread Ordering** means controlling the sequence in which multiple threads execute certain operations.

By default, thread execution order is **not guaranteed** because it is managed by the JVM and operating system scheduler.

## Example: Unordered Execution

```java
class MyThread extends Thread {
    public void run() {
        System.out.println(getName());
    }
}

public class Main {
    public static void main(String[] args) {
        new MyThread().start();
        new MyThread().start();
        new MyThread().start();
    }
}
```

Output can vary:

```text
Thread-2
Thread-0
Thread-1
```

or

```text
Thread-0
Thread-1
Thread-2
```

***

## Method 1: Using `join()`

`join()` makes one thread wait until another thread finishes.

```java
public class ThreadOrdering {

    public static void main(String[] args) throws Exception {

        Thread t1 = new Thread(() ->
                System.out.println("First"));

        Thread t2 = new Thread(() ->
                System.out.println("Second"));

        Thread t3 = new Thread(() ->
                System.out.println("Third"));

        t1.start();
        t1.join(); // Wait for t1

        t2.start();
        t2.join(); // Wait for t2

        t3.start();
    }
}
```

Output:

```text
First
Second
Third
```

***

## Method 2: Using `CountDownLatch`

Useful when one thread should start only after another completes.

```java
import java.util.concurrent.CountDownLatch;

public class Main {

    public static void main(String[] args) {

        CountDownLatch latch = new CountDownLatch(1);

        Thread t1 = new Thread(() -> {
            System.out.println("Thread 1");
            latch.countDown();
        });

        Thread t2 = new Thread(() -> {
            try {
                latch.await();
                System.out.println("Thread 2");
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        t1.start();
        t2.start();
    }
}
```

Output:

```text
Thread 1
Thread 2
```

***

## Method 3: Using `ExecutorService` (Preferred)

For production applications, use executors instead of manually creating threads.

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class Main {

    public static void main(String[] args) {

        ExecutorService executor =
                Executors.newSingleThreadExecutor();

        executor.submit(() -> System.out.println("First"));
        executor.submit(() -> System.out.println("Second"));
        executor.submit(() -> System.out.println("Third"));

        executor.shutdown();
    }
}
```

Output:

```text
First
Second
Third
```

Since a single-thread executor processes tasks sequentially, order is guaranteed.

## Interview Question: Print Threads in Order

Print:

```text
A
B
C
```

using three threads.

```java
Thread t1 = new Thread(() -> System.out.println("A"));

Thread t2 = new Thread(() -> {
    try {
        t1.join();
        System.out.println("B");
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
});

Thread t3 = new Thread(() -> {
    try {
        t2.join();
        System.out.println("C");
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
});

t1.start();
t2.start();
t3.start();
```

Output:

```text
A
B
C
```

### Key Interview Takeaway

Common ways to achieve thread ordering:

* `Thread.join()`
* `CountDownLatch`
* `CyclicBarrier`
* `Semaphore`
* `CompletableFuture`
* `ExecutorService` (single-threaded or chained tasks)

For modern Java applications, **`CompletableFuture`** and **`ExecutorService`** are generally preferred over direct thread management.
