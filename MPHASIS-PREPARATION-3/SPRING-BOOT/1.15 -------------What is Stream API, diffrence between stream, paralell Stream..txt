## What is Stream API in Java?

The **Stream API**, introduced in **Java 8**, is used to process collections of data in a **declarative and functional style**.

Instead of writing loops, you describe **what operation** should be performed on the data.

### Without Stream API

```java
List<String> names = Arrays.asList("Sunil", "Raj", "Kiran");

for (String name : names) {
    if (name.startsWith("S")) {
        System.out.println(name);
    }
}
```

### With Stream API

```java
List<String> names = Arrays.asList("Sunil", "Raj", "Kiran");

names.stream()
     .filter(name -> name.startsWith("S"))
     .forEach(System.out::println);
```

Output:

```text
Sunil
```

***

## Features of Stream API

* Functional programming style
* Supports filtering, mapping, sorting, grouping, reduction,
* Improves readability
* Supports parallel processing
* Does not modify the original collection

***

## Common Stream Operations

### Filter

```java
List<Integer> numbers = List.of(1, 2, 3, 4, 5);

numbers.stream()
       .filter(n -> n % 2 == 0)
       .forEach(System.out::println);
```

Output:

```text
2
4
```

### Map

Transform elements.

```java
numbers.stream()
       .map(n -> n * 2)
       .forEach(System.out::println);
```

Output:

```text
2
4
6
8
10
```

### Collect

```java
List<Integer> evenNumbers =
        numbers.stream()
               .filter(n -> n % 2 == 0)
               .collect(Collectors.toList());
```

***

## Stream vs Parallel Stream

### Normal Stream

```java
list.stream()
```

Processes data using **a single thread**.

### Parallel Stream

```java
list.parallelStream()
```

Processes data using **multiple threads** from the ForkJoinPool.

***

## Example

### Sequential Stream

```java
List<Integer> numbers =
        IntStream.rangeClosed(1, 10)
                 .boxed()
                 .toList();

numbers.stream()
       .forEach(i ->
            System.out.println(
                i + " : " +
                Thread.currentThread().getName()));
```

Output:

```text
1 : main
2 : main
3 : main
...
10 : main
```

***

### Parallel Stream

```java
numbers.parallelStream()
       .forEach(i ->
            System.out.println(
                i + " : " +
                Thread.currentThread().getName()));
```

Output:

```text
6 : ForkJoinPool.commonPool-worker-1
3 : main
9 : ForkJoinPool.commonPool-worker-2
...
```

Notice:

* Multiple threads are used.
* Processing order is not guaranteed.

***

## Difference Between Stream and Parallel Stream

| Feature       | Stream                    | Parallel Stream                        |
| ------------- | ------------------------- | -------------------------------------- |
| Threads Used  | Single Thread             | Multiple Threads                       |
| Execution     | Sequential                | Parallel                               |
| Order         | Preserved                 | Not guaranteed (`forEach`)             |
| Performance   | Better for small datasets | Better for large CPU-intensive tasks   |
| Thread-Safety | Simpler                   | Need caution with shared mutable state |
| Overhead      | Low                       | Higher                                 |

***

## When to Use Parallel Stream?

✅ Large datasets

✅ CPU-intensive operations

✅ Independent tasks

Example:

```java
long count = employees.parallelStream()
                      .filter(emp -> emp.getSalary() > 100000)
                      .count();
```

***

## When NOT to Use Parallel Stream?

❌ Small collections

❌ Database calls

❌ Network/API calls

❌ Operations with shared mutable variables

Bad Example:

```java
List<Integer> result = new ArrayList<>();

numbers.parallelStream()
       .forEach(result::add);
```

This may lead to race conditions.

Use:

```java
List<Integer> result =
        numbers.parallelStream()
               .collect(Collectors.toList());
```

***

## Ordering in Parallel Streams

`forEach()` does not guarantee order:

```java
List.of(1,2,3,4,5)
    .parallelStream()
    .forEach(System.out::println);
```

Possible output:

```text
3
5
1
4
2
```

To preserve order:

```java
List.of(1,2,3,4,5)
    .parallelStream()
    .forEachOrdered(System.out::println);
```

Output:

```text
1
2
3
4
5
```

***

## Interview Answer (Short)

**Stream API** is a Java 8 feature that enables functional-style processing of collections using operations such as
 `filter`, `map`, `sorted`, and `collect`.

**Stream** processes elements sequentially using a single thread, whereas **Parallel Stream** processes elements concurrently using multiple threads from the ForkJoinPool, which can improve performance for large CPU-intensive workloads but introduces thread-management overhead and ordering considerations.

### How does `parallelStream()` handle exceptions?

In a `parallelStream()`, if a thread throws an exception, the exception is propagated back to the calling thread and the stream operation typically fails.

### Example

```java
List<Integer> numbers = List.of(1, 2, 0, 4, 5);

numbers.parallelStream()
       .map(n -> 10 / n)
       .forEach(System.out::println);
```

Output:

```text
java.lang.ArithmeticException: / by zero
```

Even though multiple elements are being processed in parallel, once an exception occurs:

* The stream operation fails.
* Some tasks may already have completed.
* Other tasks may be cancelled or stop processing.
* The exception is rethrown to the caller.

***

## Catching the Exception Outside

```java
try {
    numbers.parallelStream()
           .map(n -> 10 / n)
           .forEach(System.out::println);
} catch (Exception e) {
    System.out.println("Error: " + e.getMessage());
}
```

Output:

```text
Error: / by zero
```

***

## Handling Exceptions Per Element

If you don't want one bad record to stop the entire stream, handle exceptions inside the lambda.

```java
numbers.parallelStream()
       .forEach(n -> {
           try {
               System.out.println(10 / n);
           } catch (Exception e) {
               System.out.println("Invalid value: " + n);
           }
       });
```

Output (order may vary):

```text
10
5
Invalid value: 0
2
```

***

## Exception in `map()`

A common pattern is to filter out failures:

```java
List<Integer> results =
    numbers.parallelStream()
           .map(n -> {
               try {
                   return 10 / n;
               } catch (Exception e) {
                   return null;
               }
           })
           .filter(Objects::nonNull)
           .toList();
```

***

## Important Interview Points

### 1. Order of Exception Detection is Not Deterministic

Because processing is parallel:

```java
numbers.parallelStream()
       .map(this::process)
       .forEach(System.out::println);
```

If multiple elements fail, there's no guarantee which exception you'll see first.

***

### 2. Some Elements May Be Processed Before Failure

```java
numbers.parallelStream()
       .forEach(n -> {
           if (n == 3)
               throw new RuntimeException("Error");
           System.out.println(n);
       });
```

Possible output:

```text
1
2
5
Error
```

Some records may be processed before the exception reaches the caller.

***

### 3. Checked Exceptions Are Not Allowed Directly

This won't compile:

```java
parallelStream()
    .map(file -> Files.readString(file)); // throws IOException
```

You must handle or wrap checked exceptions:

```java
parallelStream()
    .map(file -> {
        try {
            return Files.readString(file);
        } catch (IOException e) {
            throw new RuntimeException(e);
        }
    });
```

***

### Interview Answer

> `parallelStream()` propagates exceptions thrown by worker threads back to the caller. If an exception occurs, the stream operation usually terminates, although some elements may already have been processed by other threads. Since execution is parallel, the order in which exceptions are encountered is not deterministic. To prevent a single failing element from stopping the whole stream, exceptions should be handled inside the stream operation (e.g., within `map()` or `forEach()`).
