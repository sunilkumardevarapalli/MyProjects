

SOLID Principles:


1. S - Single Responsibility Principle (SRP)

	A class should have only one reason to change, meaning it should have only one responsibility.
	
2. O - Open Closed Principle (OCP)

	✅ Open for Extension

	❌ Closed for Modification

	interface INotification

		class EmailNotification extends INotification

		class SmsNotification extends INotification
	
3. L - Liskov Substitution Principle (LSP)

	A child class should be replaceable with its parent class without breaking application behavior.

	Class Bird have fly method

	but this class will not extend for Man Class , Man should not have fly method.

	Man class override the fly method should throw exception

4. I - Interface Segregation Principle (ISP)

	InterFace should have proper method Iworker have -> worker and Eat methods
	
	here ManWorker can do both but Robort will not do Eat..
	
	so two interface need to create.
