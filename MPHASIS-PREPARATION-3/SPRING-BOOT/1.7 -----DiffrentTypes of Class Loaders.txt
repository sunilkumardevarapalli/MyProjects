Different Types of Class Loaders in Java – Complete Details:

A Class Loader in Java is a part of the Java Runtime Environment (JRE) that dynamically loads Java classes 
into memory when they are required. Java uses a Class Loader Hierarchy to load classes efficiently and securely.

What is a Class Loader?

	A Class Loader is responsible for:

		Loading .class files into JVM memory
		Verifying class bytecode
		Linking classes
		Initializing classes

Java Class Loader Hierarchy:

		Bootstrap ClassLoader
				 ↑
		Extension (Platform) ClassLoader
				 ↑
		Application (System) ClassLoader
				 ↑
		Custom ClassLoader

	Java follows the Parent Delegation Model.

1. Bootstrap Class Loader

	Definition: The Bootstrap Class Loader is the root class loader. It is responsible for loading core Java classes from the JDK.

	JAVA_HOME/jre/lib
	JAVA_HOME/lib
	
	Examples:
	
		java.lang.String
		java.lang.Object
		java.lang.System
		java.util.ArrayList
		
	Checking Bootstrap Loader:

		System.out.println(String.class.getClassLoader());  output: null

	Why null?
		Because Bootstrap Class Loader is implemented in native code (C/C++) and is not a Java object.
		
2. Extension Class Loader (Java 8)

	Definition
		Loads classes from extension libraries.
		Loads from:		
		
			JAVA_HOME/jre/lib/ext
	Example
		Libraries placed in:jre/lib/ext
		
	will be loaded automatically.
	
	Checking:
	
	ClassLoader loader = javax.crypto.Cipher.class.getClassLoader();

		System.out.println(loader);
		
	Output:sun.misc.Launcher$ExtClassLoader
