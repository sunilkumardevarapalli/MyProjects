The number of threads used by a **Parallel Stream does NOT depend on the number of records (50,000)**. 
It primarily depends on the **ForkJoinPool's parallelism level**, which by default is based on the number of available CPU cores.

### Default Behavior

```java
List<Employee> employees = getEmployees();

employees.parallelStream().forEach(this::process);
```

Behind the scenes, Java uses:

```java
ForkJoinPool.commonPool()
```

The default parallelism is approximately:

```java
Runtime.getRuntime().availableProcessors() - 1
```

For example:

| CPU Cores | Parallelism |
| --------- | ----------- |
| 4         | 3 threads   |
| 8         | 7 threads   |
| 16        | 15 threads  |

So even if you have **50,000 records**, Java does **not create 50,000 threads**.

Instead:

* Data is split into chunks.
* A small number of worker threads process those chunks.
* Threads reuse work via the Fork/Join framework.

***

### Check Available Processors

```java
System.out.println(
    Runtime.getRuntime().availableProcessors()
);
```

Example output:

```text
8
```

Then Parallel Stream will typically use about:

```text
7 worker threads + main thread
```

***

### See Threads in Action

```java
IntStream.range(1, 20)
         .parallel()
         .forEach(i ->
             System.out.println(
                 i + " -> " +
                 Thread.currentThread().getName()
             ));
```

Possible output:

```text
1 -> ForkJoinPool.commonPool-worker-1
2 -> ForkJoinPool.commonPool-worker-3
3 -> main
4 -> ForkJoinPool.commonPool-worker-2
...
```

***

### For 50,000 Records Example

Suppose:

```java
List<Employee> employees = // 50,000 records

employees.parallelStream()
         .map(this::calculateSalary)
         .collect(Collectors.toList());
```

On an 8-core machine:

* Records: 50,000
* Threads: \~7 worker threads
* Each thread may process \~7,000 records (not exactly equal)
* Work is dynamically balanced

So:

```text
50,000 Records ≠ 50,000 Threads
```

It is more like:

```text
50,000 Records
        ↓
Split into multiple tasks
        ↓
7-15 Worker Threads (depending on CPU)
        ↓
Results merged
```

***

### Can We Increase the Number of Threads?

Yes.

```java
ForkJoinPool pool = new ForkJoinPool(20);

pool.submit(() ->
    employees.parallelStream()
             .forEach(this::process)
).get();
```

This creates a custom pool with 20 worker threads.

***

### Interview Answer

> Parallel Stream uses the `ForkJoinPool.commonPool()` and the number of threads is generally based on the available CPU cores,
 not the number of records. For 50,000 records on an 8-core machine, typically around 7 worker threads process the data in parallel 
 by dividing the records into smaller tasks and executing them concurrently.
