What is Angular Architecture?

	Angular architecture is the structure of an Angular application.
	It mainly contains:

		Component
		Metadata
		Template
		Service
		Data Binding
		Directives
		Dependency Injection

1) Component

	It sends data to the template and receives events from the template.
	
	A Component is a TypeScript class.

		It acts like a controller on the client side.
		It contains:
			Variables
			Methods
			UI logic
		It sends data to the template  and receives events from the template.
		It handles user actions from the template.

	Example:

		import { Component } from '@angular/core';

		@Component({
		  selector: 'app-user',
		  templateUrl: './user.component.html'
		})
		export class UserComponent {

		  name = "Sunil";

		  changeName() {
			this.name = "Kumar";
		  }
		}
	
2) Metadata

	Metadata is information about the component.
	In Angular, metadata is written using decorators like:@Component()
	
	It is written inside the @Component({...}) decorator.
	
	It tells Angular:
		the component name (selector)
		which HTML file to use (templateUrl)
		which CSS file to use (styleUrls)
		
	Metadata tells Angular:

		Which component to load
		Which HTML file belongs to the component
		Which CSS file belongs to the component
		Component selector name
		
	Example:
	
		@Component({
		  selector: 'app-user',
		  templateUrl: './user.component.html',
		  styleUrls: ['./user.component.css']
		})
		
	Explanation:

			selector → HTML tag name
			templateUrl → HTML file
			styleUrls → CSS file

	So metadata decides which component is linked to which template.	
	
	Metadata connects: Component ↔ Template

3) Template

	Template is nothing but an HTML file.
	Purpose:
			Displays data from component
			Sends events/actions to component
	Example:
			<h1>{{name}}</h1>
			<button (click)="changeName()">
			  Change
			</button>
	Here:
		Component → Template:{{name}}	
			
		Template → Component:(click)="changeName()"
		
	A template is the HTML file.
	It displays data from the component.
	It can also send user actions back to the component using events like click, input, submit, etc.	
		
	Example: 
		<h2>{{ name }}</h2>
		<button (click)="changeName()">Change Name</button>	
	Here:
		{{ name }} = component → template
		(click)="changeName()" = template → component
	
4) Service

	A Service is a TypeScript class used for:

		Business logic
		REST API calls
		Database communication
		Shared logic
		Reusable code
		
	We usually inject it into a component using dependency injection.

	Example:

		@Injectable({ providedIn: 'root' })
		export class UserService {
		  getUsers() {
			return ['A', 'B', 'C'];
		  }
		}	
		
		Injecting service into component: constructor(private service: UserService) {}
		
5) Data binding

	Data binding means communication between:

	Component ↔ Template

	Angular supports 4 types of data binding. a) Interpolation	b) Property Binding	c) Event Binding	d) Two-Way Data Binding

	Angular supports communication between component and template:

		Interpolation:	 Component → Template 		: {{ value }}
			
		Property binding:Component → Template	: [value]="name"
			
		Event binding:	 Template → Component		: (click)="save()"
			
		Two-way binding: Both ways or	Both directions: [(ngModel)]="name"  or <input [(ngModel)]="name">
				
	
6) Directives 

	Directives are used to manipulate DOM elements.

	Common directives:

		Angular generally handles DOM through templates, bindings, directives, and lifecycle hooks.
		Direct DOM access is possible, but Angular prefers working through its own rendering system.
	
	Example tools:
	
		*ngIf: Used for conditional display. Ex:	<div *ngIf="isVisible">
		
		*ngFor: Used for looping.  <li *ngFor="let e of employees">
		
		ngClass: Used for dynamic CSS classes.	<div [ngClass]="myClass">
		
		[class.active]
		[style.color]
		
7) Dependency Injection (DI)

	Dependency Injection means Angular automatically creates and injects objects.
	
	Example:

		constructor(private service: UserService)

	Angular automatically creates UserService object and injects it into component.
	
Complete Angular Flow:

	User Action
		 ↓
	Template (HTML)
		 ↓
	Component (TypeScript)
		 ↓
	Service
		 ↓
	REST API / Backend
		 ↓
	Response from Backend
		 ↓
	Component updates data
		 ↓
	Template updates automatically

DOM Manipulation in Angular

	Angular manipulates DOM using:

		Directives
		Data binding
		Change detection

	Examples:

		*ngIf
		*ngFor
		ngClass
		ngStyle

	Angular automatically updates DOM whenever component data changes.

Simple flow

	Component contains data and logic.
	Metadata connects component to template.
	Template shows the UI.
	Service contains reusable business logic and API calls.
	Data binding connects template and component.
	DOM is updated by Angular automatically when data changes.

One-line correction of your statement

	Instead of saying “template will send data to component”, it is better to say:

	Template sends user events to the component, and component sends data to the template.

Very Short Interview Answer

	Angular architecture contains Component, Template, Metadata, Service, and Data Binding.

	Component is a TypeScript class that handles UI logic.

	Template is an HTML file used to display data.

	Metadata connects component with template using @Component decorator.

	Services contain business logic and REST API calls.

	Data binding enables communication between component and template.

	Angular also uses dependency injection and directives to manipulate DOM dynamically.
