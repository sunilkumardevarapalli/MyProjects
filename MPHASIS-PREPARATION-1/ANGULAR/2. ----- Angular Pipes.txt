

# Pipes in Angular — Complete Detailed Explanation (12+ Years Experience)

	## What is a Pipe in Angular?

		A Pipe is used to:

			* Transform data in the template
			* Format data before displaying in UI

		Pipe takes input data and returns transformed output.

	Suppose component has:

		name = "sunil";
		salary = 50000;
		today = new Date();

	Without pipes:

		{{name}}
		{{salary}}
		{{today}}

	With pipes:

		{{name | uppercase}}
		{{salary | currency}}
		{{today | date}}

	Output:

		SUNIL
		₹50,000
		May 22, 2026


# Why Pipes?

	Pipes improve:

		* Readability
		* Reusability
		* Clean templates
		* Data formatting

	Used for:

		* Date formatting
		* Currency formatting
		* Text transformation
		* Filtering
		* Sorting
		* Custom business formatting

# Syntax of Pipe

	{{ value | pipeName }}

Multiple pipes:

	{{ value | pipe1 | pipe2 }}

Pipe with parameters:

	{{ amount | currency:'INR':'symbol':'1.2-2' }}

# Built-in Pipes in Angular

	Angular provides many built-in pipes.

---

# 1) UpperCasePipe

	Transforms text to uppercase.

		{{ name | uppercase }}

	Output:

		SUNIL
		
# 2) LowerCasePipe

	Transforms text to lowercase. 

	{{ name | lowercase }}

# 3) TitleCasePipe

	First letter capital. 
	{{ name | titlecase }} 

	Output:
		Sunil Kumar

# 4) DatePipe

	Formats date.

	{{ today | date }}

	Custom formats: 

	{{ today | date:'dd/MM/yyyy' }}
	{{ today | date:'fullDate' }}
	{{ today | date:'shortTime' }}


# 5) CurrencyPipe

	Formats currency.

	{{ salary | currency }}


	Specific currency: 

	{{ salary | currency:'INR' }}
	{{ salary | currency:'USD' }}

# 6) DecimalPipe

	Formats numbers. 
		{{ num | number:'1.2-2' }} 

	Meaning:

		* Minimum integer digits = 1
		* Minimum decimal digits = 2	
		* Maximum decimal digits = 2

# 7) PercentPipe

	Formats percentage. 
	{{ 0.75 | percent }} 

	Output: 
	75% 

# 8) SlicePipe

	Slices array/string. 
	{{ name | slice:0:3 }} 

# 9) JsonPipe

	Converts object to JSON.

	Mostly used for debugging. 
	{{ employee | json }} 

# 10) AsyncPipe (Very Important)

	Used with:

	* Observable
	* Promise

	Automatically:

	* subscribes
	* unsubscribes

	Example: 
	
	users$ = this.http.get('/users'); 
	
	<div *ngFor="let u of users$ | async">
	  {{u.name}}
	</div> 

# Why AsyncPipe Important?

	Without async pipe: 
	this.service.getUsers().subscribe(data => {
	   this.users = data;
	}); 
	Need manual unsubscribe.

	With async pipe:

	* Angular handles subscription automatically
	* Prevents memory leaks

	Very important in enterprise applications.

# Custom Pipes

	Angular allows creating custom pipes.

---

# Creating Custom Pipe

	Command: 
		ng generate pipe salary

	OR 
		ng g p salary

	Generated: 

		import { Pipe, PipeTransform } from '@angular/core';

		@Pipe({
		  name: 'salary'
		})
		export class SalaryPipe implements PipeTransform {

		  transform(value: number): string {
			return "₹" + value;
		  }
		} 
	Usage:
	 
		{{50000 | salary}} 

	Output: 
		₹50000 

	# PipeTransform Interface

		Every custom pipe implements:  PipeTransform 

	Method: transform(value: any): any 

	This method receives input and returns transformed output.

---

# Parameterized Custom Pipe

	Example: 
		transform(value: number, symbol: string): string {
		   return symbol + value;
		} 
	Usage: 
		{{5000 | salary:'₹'}} 

# Pure Pipes vs Impure Pipes (Very Important)

		This is one of the top interview questions.

---

# Pure Pipe (Default)

	Runs only when:

		* input reference changes
		* primitive value changes

	Better performance.

		Example: 

			@Pipe({
			  name: 'test',
			  pure: true
			}) 
		Default: 
			pure = true 

# Impure Pipe

	Runs during every change detection cycle.

		Example: 
			@Pipe({
			  name: 'test',
			  pure: false
			}) 

# Real-Time Example

	Suppose: employees.push(newEmployee); 

	Reference not changed.

	Pure pipe:

		* may NOT execute

	Impure pipe:

		* executes every time

---

# Performance Impact

	Impure pipes:

		* expensive
		* reduce performance
		* avoid unless necessary

	In large enterprise apps:

		* prefer pure pipes
		* use trackBy
		* use OnPush change detection

---

# AsyncPipe Internals (Senior-Level)

	AsyncPipe internally:

		* subscribes to Observable/Promise
		* listens for emitted values
		* triggers change detection
		* automatically unsubscribes on destroy

	Internally uses:

		* ChangeDetectorRef
		* Subscription management

---

# Pipe Chaining

Multiple pipes together.
 
	{{ name | uppercase | slice:0:4 }} 
	Execution:

	1. uppercase
	2. slice

---

# Pipe Operator Priority

	Pipe has lower priority.

	Use parentheses when needed.

	Example:

	 
	{{ (amount + tax) | currency }}
	 

# Internationalization (i18n) with Pipes

	DatePipe and CurrencyPipe support locale.

	Example:

	 
	providers: [
	  { provide: LOCALE_ID, useValue: 'fr' }
	]

# Best Practices (12+ Years Experience)

## Use Pure Pipes Whenever Possible

Improves performance.

---

## Avoid Heavy Logic Inside Pipes

	Bad:

	database calls
	HTTP calls
	large loops 

	Pipe should only transform data.

---

## Do NOT Use Pipes for Business Logic

	Business logic belongs to:

	* Services
	* Components

	Not pipes.

---

## Use AsyncPipe Instead of Manual Subscribe

	Good:

	users$ | async

	Avoid:

	subscribe()
	unsubscribe()

## Prefer Memoization for Expensive Transformations

	Large datasets:

	* use caching
	* memoization

# Pipe Lifecycle

	For pure pipe:

	* executes only on input change

	For impure pipe:

	* executes every CD cycle

---

# Difference Between Pipe and Directive

	| Pipe                  | Directive          |
	| --------------------- | ------------------ |
	| Transforms data       | Manipulates DOM    |
	| Used in interpolation | Used as attributes |
	| Output formatting     | UI behavior        |

	Example:

	Pipe: 
{{name | uppercase}} 

Directive:

<div *ngIf="isVisible">

# Final Senior-Level Summary

	> Pipes in Angular are used for transforming and formatting data in templates.
	>
	> Angular provides built-in pipes like date, currency, uppercase, lowercase, async, json, etc.
	>
	> Custom pipes can be created using PipeTransform interface.
	>
	> Pipes are categorized into pure and impure pipes.
	>
	> Pure pipes execute only when input reference changes and provide better performance.
	>
	> AsyncPipe is heavily used in enterprise applications because it automatically manages subscriptions and prevents memory leaks.
	>
	> Pipes should only contain presentation logic, not business logic.
