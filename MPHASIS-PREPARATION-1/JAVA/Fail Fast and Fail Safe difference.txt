## Fail-Fast vs Fail-Safe Iterator in Java

| Feature                 | Fail-Fast                                | Fail-Safe                               |
| ----------------------- | ---------------------------------------- | --------------------------------------- |
| Iterates on             | Original collection                      | Copy/Snapshot of collection             |
| Concurrent Modification | Throws `ConcurrentModificationException` | No exception                            |
| Reflects latest changes | No                                       | No (iterates over snapshot)             |
| Thread-safe             | No                                       | Yes                                     |
| Performance             | Faster                                   | Slight overhead due to copy             |
| Package                 | `java.util`                              | `java.util.concurrent`                  |
| Examples                | ArrayList, HashMap, HashSet              | CopyOnWriteArrayList, ConcurrentHashMap |

---

## 1. Fail-Fast Iterator

A fail-fast iterator throws `ConcurrentModificationException` if the collection is structurally modified after the iterator is created (except through the iterator itself).

### Example

```java
List<String> list = new ArrayList<>();

list.add("A");
list.add("B");
list.add("C");

Iterator<String> itr = list.iterator();

while (itr.hasNext()) {
    String value = itr.next();

    if (value.equals("B")) {
        list.add("D"); // Structural modification
    }

    System.out.println(value);
}
```

### Output

```text
A
B
Exception in thread "main"
java.util.ConcurrentModificationException
```

### Why?

* Iterator maintains `expectedModCount`.
* Collection maintains `modCount`.
* If `modCount != expectedModCount`, iterator throws `ConcurrentModificationException`.

---

## Safe Modification Using Iterator

```java
List<String> list = new ArrayList<>();

Iterator<String> itr = list.iterator();

while (itr.hasNext()) {
    String value = itr.next();

    if (value.equals("B")) {
        itr.remove(); // Safe
    }
}
```

## 2. Fail-Safe Iterator

Fail-safe iterators iterate over a **snapshot (copy)** of the collection.

Modifications do not affect the iterator and no exception is thrown.

### Example with CopyOnWriteArrayList

```java
CopyOnWriteArrayList<String> list =
        new CopyOnWriteArrayList<>();

list.add("A");
list.add("B");
list.add("C");

Iterator<String> itr = list.iterator();

while (itr.hasNext()) {
    String value = itr.next();

    if (value.equals("B")) {
        list.add("D");
    }

    System.out.println(value);
}
```

### Output

```text
A
B
C
```

No exception occurs.

The newly added `"D"` is not visible because the iterator traverses the snapshot created when the iterator was obtained.

---

## ConcurrentHashMap Example

```java
ConcurrentHashMap<Integer, String> map =
        new ConcurrentHashMap<>();

map.put(1, "A");
map.put(2, "B");

for (Integer key : map.keySet()) {
    map.put(3, "C");
    System.out.println(key);
}
```

No `ConcurrentModificationException` occurs.

## Internal Mechanism

### Fail-Fast

```text
modCount = 5
expectedModCount = 5

Collection modified

modCount = 6
expectedModCount = 5

=> ConcurrentModificationException
```

---

### Fail-Safe

```text
Original Collection
      ↓
Snapshot Copy
      ↓
Iterator traverses copy
      ↓
Changes to original collection don't affect iteration
```

## Real-Time Scenarios

### Fail-Fast

Used in:

* `ArrayList`
* `HashMap`
* `HashSet`
* `LinkedList`

Best when:

* Single-threaded applications
* Modifications during iteration are not expected

---

### Fail-Safe

Used in:

* `CopyOnWriteArrayList`
* `ConcurrentHashMap`
* Concurrent collections

Best when:

* Multiple threads read and write simultaneously
* High-concurrency applications
* Caches, sessions, message processing

---

## Interview Answer (10+ Years Experience)

> A fail-fast iterator works on the original collection and detects structural modifications using `modCount` and `expectedModCount`. If the collection is modified during iteration, it throws `ConcurrentModificationException`. A fail-safe iterator operates on a snapshot or weakly consistent view of the collection, allowing concurrent modifications without throwing exceptions. In multithreaded applications, I prefer concurrent collections like `ConcurrentHashMap` or `CopyOnWriteArrayList` instead of synchronized collections or manually handling concurrent modifications.
