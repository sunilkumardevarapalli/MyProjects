If you're preparing for Java 17/21 interviews, here are the most common **Record** and **Sealed Class** examples.

---

# 1. Simple Record


		public record Employee(int id, String name, double salary) {}


		Usage:


		Employee emp = new Employee(101, "John", 50000);

		System.out.println(emp.id());
		System.out.println(emp.name());
		System.out.println(emp.salary());


		Output


		101
		John
		50000.0


		Records automatically generate:

		* Constructor
		* Getters (`id()`, `name()`)
		* `equals()`
		* `hashCode()`
		* `toString()`

---

# 2. Record with Custom Constructor


		public record Employee(int id, String name) {

			public Employee {
				if (id <= 0)
					throw new IllegalArgumentException("Invalid Id");

				name = name.toUpperCase();
			}
		}


		Usage


		Employee emp = new Employee(101, "john");

		System.out.println(emp);


		Output


		Employee[id=101, name=JOHN]


---

# 3. Record with Methods


		public record Rectangle(double length, double width) {

			public double area() {
				return length * width;
			}

			public double perimeter() {
				return 2 * (length + width);
			}
		}


		Usage


		Rectangle r = new Rectangle(10, 5);

		System.out.println(r.area());
		System.out.println(r.perimeter());


		Output


		50.0
		30.0


		---

# 4. Record Implementing Interface

		interface Shape {
			double area();
		}

		public record Circle(double radius) implements Shape {

			@Override
			public double area() {
				return Math.PI * radius * radius;
			}
		}


		Usage


		Shape s = new Circle(5);

		System.out.println(s.area());


		---

# 5. Nested Record


		public class Company {

			public record Employee(int id, String name) {
			}
		}


		Usage


		Company.Employee emp =
				new Company.Employee(101, "John");


		---

# 6. Generic Record


		public record Response<T>(String message, T data) {
		}


		Usage


		Response<String> r =
				new Response<>("Success", "Employee Saved");

		System.out.println(r);


		---

# 7. Record with Static Members


		public record Employee(int id, String name) {

			static String company = "ABC";

			public static void printCompany() {
				System.out.println(company);
			}
		}


		Usage


		Employee.printCompany();


		---

# 8. Sealed Class

		public sealed class Vehicle
				permits Car, Bike {
		}

		Allowed subclasses

		public final class Car extends Vehicle {
		}

		public final class Bike extends Vehicle {
		}

		Only `Car` and `Bike` can extend `Vehicle`.
		---
# 9. Sealed Interface


		public sealed interface Shape
				permits Circle, Rectangle {
		}


		Implementations


		public final class Circle implements Shape {
		}

		public final class Rectangle implements Shape {
		}


		---

# 10. Sealed Class + Non-Sealed Class


public sealed class Animal
        permits Dog {

}

public non-sealed class Dog extends Animal {

}

class GermanShepherd extends Dog {

}


`Dog` is `non-sealed`, so any class can extend it.

---

# 11. Sealed Class + Final Class


public sealed class Payment
        permits CreditCard, UPI {

}

public final class CreditCard extends Payment {

}

public final class UPI extends Payment {

}


No other subclasses are allowed.

---

# 12. Sealed Class + Sealed Subclass


public sealed class Vehicle
        permits Car {

}

public sealed class Car extends Vehicle
        permits BMW, Audi {

}

public final class BMW extends Car {

}

public final class Audi extends Car {

}


Hierarchy


Vehicle
   |
  Car
 /   \
BMW  Audi

# 13. Record + Sealed Interface (Most Popular Interview Example)


public sealed interface Employee
        permits PermanentEmployee, ContractEmployee {
}



public record PermanentEmployee(
        int id,
        String name,
        double salary)
        implements Employee {
}



public record ContractEmployee(
        int id,
        String name,
        int hours)
        implements Employee {
}


Usage


Employee e1 =
        new PermanentEmployee(1, "John", 60000);

Employee e2 =
        new ContractEmployee(2, "Mike", 120);


---

# 14. Pattern Matching with Record and Sealed Interface (Java 21)


public sealed interface Shape
        permits Circle, Rectangle {
}

public record Circle(double radius)
        implements Shape {
}

public record Rectangle(double length, double width)
        implements Shape {
}



static double area(Shape shape) {

    return switch (shape) {

        case Circle(double r) ->
                Math.PI * r * r;

        case Rectangle(double l, double w) ->
                l * w;
    };
}


Usage


System.out.println(area(new Circle(5)));
System.out.println(area(new Rectangle(10, 5)));

## Interview Questions

1. What is a record?
2. Why are records immutable?
3. Can records extend another class? (**No**, they implicitly extend `java.lang.Record`.)
4. Can records implement interfaces? (**Yes**.)
5. Can records have constructors? (**Yes**, including compact constructors.)
6. What is a sealed class?
7. What are `final`, `sealed`, and `non-sealed` subclasses?
8. Why use sealed classes? (To restrict inheritance and model a fixed set of types.)
9. Can records be used with sealed interfaces? (**Yes**, and this is a very common Java 17/21 pattern.)
10. What are the benefits of combining records and sealed classes? (Immutable data + restricted hierarchies + exhaustive pattern matching.)
