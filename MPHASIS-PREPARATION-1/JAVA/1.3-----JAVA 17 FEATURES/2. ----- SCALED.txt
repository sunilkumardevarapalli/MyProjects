Below are the **valid declarations and implementations** of **sealed classes**, **sealed interfaces**, and their combinations with **records** in Java 17/21.

---

# 1. Sealed Class → Final Class

```java
public sealed class Vehicle
        permits Car, Bike {
}

public final class Car extends Vehicle {
}

public final class Bike extends Vehicle {
}
```

✅ `Car` and `Bike` cannot be extended.

---

# 2. Sealed Class → Sealed Class

```java
public sealed class Vehicle
        permits Car {
}

public sealed class Car extends Vehicle
        permits BMW, Audi {
}

public final class BMW extends Car {
}

public final class Audi extends Car {
}
```

Hierarchy

```
Vehicle
   |
  Car
 /   \
BMW  Audi
```

---

# 3. Sealed Class → Non-Sealed Class

```java
public sealed class Animal
        permits Dog {
}

public non-sealed class Dog extends Animal {
}

class GermanShepherd extends Dog {
}

class Labrador extends Dog {
}
```

Since `Dog` is `non-sealed`, anyone can extend it.

---

# 4. Sealed Interface → Final Classes

```java
public sealed interface Shape
        permits Circle, Rectangle {
}

public final class Circle implements Shape {
}

public final class Rectangle implements Shape {
}
```

---

# 5. Sealed Interface → Sealed Class

```java
public sealed interface Vehicle
        permits Car {
}

public sealed class Car
        implements Vehicle
        permits BMW {
}

public final class BMW extends Car {
}
```

---

# 6. Sealed Interface → Non-Sealed Class

```java
public sealed interface Animal
        permits Dog {
}

public non-sealed class Dog
        implements Animal {
}

class Labrador extends Dog {
}
```

---

# 7. Sealed Interface → Records

```java
public sealed interface Employee
        permits PermanentEmployee,
                ContractEmployee {
}

public record PermanentEmployee(
        int id,
        String name,
        double salary)
        implements Employee {
}

public record ContractEmployee(
        int id,
        String name,
        int hours)
        implements Employee {
}
```

Records are implicitly `final`, so they satisfy the requirement for permitted subclasses.

---

# 8. Sealed Class → Records ❌

```java
public sealed class Employee
        permits PermanentEmployee {
}
```

```java
public record PermanentEmployee(
        int id,
        String name)
        extends Employee {
}
```

❌ **Compilation Error**

Reason:

A record cannot extend a class because every record already extends `java.lang.Record`.

---

# 9. Sealed Interface → Mix of Class and Record

```java
public sealed interface Payment
        permits CreditCard, UPI {
}

public final class CreditCard
        implements Payment {
}

public record UPI(
        String upiId)
        implements Payment {
}
```

```java
public record Email(
```

# 14. Generic Sealed Interface + Records

```java

        permits Success, Error {
}

public record Success<T>(
        T data)
        implements Result<T> {
}

public record Error<T>(
```

# 15. Pattern Matching with Sealed Interface + Records (Java 21)

```java
public sealed interface Shape
        permits Circle, Rectangle {
}

public record Circle(double radius)
        implements Shape {
}

public record Rectangle(
        double length,
        double width)
        implements Shape {
}
```

```java
public static double area(Shape shape) {

    return switch (shape) {

        case Circle(double r) ->
                Math.PI * r * r;

        case Rectangle(double l, double w) ->
                l * w;
    };
}
```

# Rules for Sealed Types

A direct subclass or implementing class of a `sealed` type **must** be declared as one of the following:

1. `final`
2. `sealed`
3. `non-sealed`

Example:

```java
public sealed class A permits B, C, D {
}

public final class B extends A {
}

public sealed class C extends A permits E {
}

public non-sealed class D extends A {
}

public final class E extends C {
}
```

# Summary Table

| Parent           | Child            | Valid? | Reason                         |
| ---------------- | ---------------- | ------ | ------------------------------ |
| Sealed Class     | Final Class      | ✅      | Allowed                        |
| Sealed Class     | Sealed Class     | ✅      | Allowed                        |
| Sealed Class     | Non-Sealed Class | ✅      | Allowed                        |
| Sealed Interface | Final Class      | ✅      | Allowed                        |
| Sealed Interface | Sealed Class     | ✅      | Allowed                        |
| Sealed Interface | Non-Sealed Class | ✅      | Allowed                        |
| Sealed Interface | Record           | ✅      | Records are implicitly `final` |
| Sealed Class     | Record           | ❌      | Records cannot extend classes  |
| Sealed Interface | Record + Class   | ✅      | Allowed                        |
| Sealed Interface | Multiple Records | ✅      | Allowed                        |
| Sealed Interface | Generic Records  | ✅      | Allowed                        |

For interviews, remember these key points:

* A **record cannot extend a class**, so it **cannot** be a permitted subclass of a sealed class.
* A **record can implement a sealed interface**, making **sealed interface + records** the idiomatic combination in Java 17 and later.
* Every direct child of a sealed type must be declared `final`, `sealed`, or `non-sealed`.
