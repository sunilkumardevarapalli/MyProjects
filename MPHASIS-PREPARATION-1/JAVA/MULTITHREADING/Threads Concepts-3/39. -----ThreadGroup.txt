## Example 1: Basic ThreadGroup
	
public class ThreadGroupDemo {

    public static void main(String[] args) {

        ThreadGroup group =  new ThreadGroup("EmployeeGroup");

        Runnable task = () -> {

            System.out.println( Thread.currentThread().getName() + " running");
        };

        Thread t1 =new Thread(group, task, "Employee-1");

        Thread t2 = new Thread(group,task,"Employee-2");

        t1.start();
        t2.start();

        System.out.println("Group Name : "+ group.getName());

        System.out.println("Active Count : " + group.activeCount());
    }
}

# invokeAll()

## What is invokeAll()?

Used when you have multiple `Callable` tasks.

List<Future<String>> futures = executor.invokeAll(tasks);

	Submit all tasks
	Execute tasks
	Wait for all tasks
	Return List<Future>

# Example 2: invokeAll() with 5 Callable Tasks


import java.util.*;
import java.util.concurrent.*;

public class InvokeAllDemo {

    public static void main(String[] args)
            throws Exception {

        ExecutorService ex = Executors.newFixedThreadPool(3);

        List<Callable<String>> tasks = new ArrayList<>();

        for(int i=1;i<=5;i++) {

            int id = i;

            tasks.add(() -> {

                Thread.sleep(2000);
                return "Customer-" + id+ " Processed";
            });
        }
        List<Future<String>> futures = ex.invokeAll(tasks);

        for(Future<String> future : futures) {

            System.out.println(future.get());
        }
        ex.shutdown();
    }
}

Customer-2 Processed
Customer-3 Processed
Customer-4 Processed

# Example 3: ThreadGroup + ExecutorService + invokeAll

Although they don't directly interact, you can use both in the same application.


import java.util.*;
import java.util.concurrent.*;

public class EmployeeProcessor {

    public static void main(String[] args)
            throws Exception {

        ThreadGroup group =
                new ThreadGroup("EmployeeGroup");

        ExecutorService ex =
                Executors.newFixedThreadPool(3);

        List<Callable<String>> tasks =
                Arrays.asList(

                        () -> {
                            Thread.currentThread()
                                    .setName("EMP-1");
                            return "Employee-1 Done";
                        },

                        () -> {
                            Thread.currentThread()
                                    .setName("EMP-2");
                            return "Employee-2 Done";
                        },

                        () -> {
                            Thread.currentThread()
                                    .setName("EMP-3");
                            return "Employee-3 Done";
                        }
                );

        List<Future<String>> futures =
                ex.invokeAll(tasks);

        for(Future<String> future : futures) {

            System.out.println(
                    future.get());
        }

        ex.shutdown();
    }
}

## submit()

	Future<String> future =ex.submit(task);

	One task at a time.

## invokeAll()

		List<Future<String>> futures = ex.invokeAll(tasks);

		Many tasks together.

		Waits for all.

# Interview Answer

> `ThreadGroup` is a legacy Java API used to logically group multiple threads for monitoring and bulk operations such as interruption.
 `invokeAll()` belongs to `ExecutorService` and is used to execute a collection of `Callable` tasks concurrently. 
 It blocks until all tasks complete and returns a `List<Future<?>>`. 
 In modern enterprise applications, `ExecutorService`, `Callable`, `Future`, and `invokeAll()`
 are preferred over direct thread management and ThreadGroup.
	
---- In above program Thread Group is added

Correct. In your example, the `ThreadGroup` is created but **never used**.


ThreadGroup group = new ThreadGroup("EmployeeGroup");


The threads created by:


ExecutorService ex = Executors.newFixedThreadPool(3);

belong to the executor's default thread factory, which creates threads in the current thread's group 
(typically the `main` thread group), **not** in `EmployeeGroup`.

## Why?

`ExecutorService` creates its own worker threads using a `ThreadFactory`.

Since you didn't provide a custom `ThreadFactory`, the executor ignores your `ThreadGroup`.

## Solution: Custom ThreadFactory

import java.util.*;
import java.util.concurrent.*;

public class EmployeeProcessor {

    public static void main(String[] args) throws Exception {

        ThreadGroup group = new ThreadGroup("EmployeeGroup");

        ThreadFactory factory = r ->  new Thread(  group, r,  "EMP-" + System.nanoTime());

        ExecutorService ex = Executors.newFixedThreadPool(  3,factory);

        List<Callable<String>> tasks =
                Arrays.asList(
                        () -> {
                            Thread t = Thread.currentThread();

                            return t.getName()  + " | Group=" + t.getThreadGroup().getName();
                        },

                        () -> {
                            Thread t = Thread.currentThread();

                            return t.getName()  + " | Group=" + t.getThreadGroup().getName();
                        },

                        () -> {
                            Thread t = Thread.currentThread();

                            return t.getName()  + " | Group=" + t.getThreadGroup().getName();
                        }
                );

        List<Future<String>> futures = ex.invokeAll(tasks);

        for (Future<String> future : futures) {
            System.out.println(future.get());
        }

        System.out.println( "\nActive Threads in Group: " + group.activeCount());

        ex.shutdown();
    }
}

### Key Interview Point

❌ This does **not** add executor threads to the group:


ThreadGroup group =
        new ThreadGroup("EmployeeGroup");

ExecutorService ex =
        Executors.newFixedThreadPool(3);


✅ This does:


ExecutorService ex =
        Executors.newFixedThreadPool(
                3,
                r -> new Thread(
                        group,
                        r));


Because `ThreadGroup` can only be associated with a thread **when the thread is created**, 
you must use a custom `ThreadFactory` with `ExecutorService`.
