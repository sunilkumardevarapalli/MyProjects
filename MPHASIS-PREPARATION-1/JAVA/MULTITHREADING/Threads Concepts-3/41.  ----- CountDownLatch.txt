import java.util.concurrent.CountDownLatch;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class CountDownLatchDemo {

    public static void main(String[] args)
            throws InterruptedException {

        ExecutorService ex = Executors.newFixedThreadPool(3);

        CountDownLatch latch =new CountDownLatch(3);

        for (int i = 1; i <= 3; i++) {

            final int taskId = i;

            ex.submit(() -> {

                try {

                    System.out.println(  "Task-" + taskId +" started by " + Thread.currentThread().getName());

                    Thread.sleep(2000);

                    System.out.println( "Task-" + taskId +" completed by " +Thread.currentThread().getName());

                } catch (InterruptedException e) {

                    Thread.currentThread().interrupt();

                    System.out.println(  "Task-" + taskId +" interrupted");

                } finally {

                    System.out.println("Task-" + taskId + " counting down");

                    latch.countDown();
                }
            });
        }

        System.out.println( "Main Thread waiting...");

        latch.await();

        System.out.println("All Tasks Finished");

        ex.shutdown();
    }
}

# Interview Answers

### Why use CountDownLatch with ExecutorService?

	> ExecutorService executes tasks asynchronously in a thread pool, while CountDownLatch allows the main thread to wait until all 
	submitted tasks complete.

### How should exceptions be handled?

	> Always place `countDown()` inside a `finally` block. Otherwise, exceptions may prevent the count from reaching zero and cause 
	waiting threads to block indefinitely.

### What happens if `countDown()` is not called?

	> The latch count never reaches zero, so threads waiting on `await()` may block forever unless a timed `await()` is used.

> `CountDownLatch` is a synchronization utility that allows one or more threads to wait until a set of operations completes. 
It is initialized with a count value. Worker threads call `countDown()` when their work finishes, reducing the count. 
Threads calling `await()` remain blocked until the count becomes zero. 

It is commonly used for application startup, service initialization, parallel processing, report generation, and API aggregation. 

A `CountDownLatch` 
is a one-time-use synchronization aid and cannot be reset or reused.

If you want a **complete Java program** demonstrating how to avoid infinite waiting with `CountDownLatch.await(timeout, TimeUnit)
` and handle timeout scenarios, here is a full example:

```java
import java.util.concurrent.CountDownLatch;
import java.util.concurrent.TimeUnit;

public class CountDownLatchTimeoutExample {

    public static void main(String[] args) {

        CountDownLatch latch = new CountDownLatch(3);

        // Task 1
        new Thread(() -> {
            try {
                Thread.sleep(1000);
                System.out.println("Task 1 Completed");
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            } finally {
                latch.countDown();
            }
        }).start();

        // Task 2
        new Thread(() -> {
            try {
                Thread.sleep(2000);
                System.out.println("Task 2 Completed");
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            } finally {
                latch.countDown();
            }
        }).start();

        // Task 3 (takes longer than timeout)
        new Thread(() -> {
            try {
                Thread.sleep(7000);
                System.out.println("Task 3 Completed");
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            } finally {
                latch.countDown();
            }
        }).start();

        try {
            System.out.println("Waiting for all tasks...");

            boolean success =
                    latch.await(
                            5,
                            TimeUnit.SECONDS);

            if (success) {

                System.out.println(
                        "All Tasks Completed");

            } else {

                System.out.println(
                        "Timeout Occurred");
            }

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            System.out.println("Main thread interrupted");
        }
    }
}
```

### Sample Output

```text
Waiting for all tasks...
Task 1 Completed
Task 2 Completed
Timeout Occurred
Task 3 Completed
```

### Key Point

Using:

```java
latch.await(5, TimeUnit.SECONDS);
```

prevents the main thread from waiting forever. It returns:

* `true` → all tasks finished before the timeout.
* `false` → timeout occurred before all tasks completed.

This is a recommended practice whenever there is a possibility that worker threads may hang, fail, or take longer than expected.

# Can CountDownLatch Be Reused?

❌ No.

Once count reaches:

```text
0
```

latch is finished.

There is no:

```java
reset()
```

method.

# CountDownLatch vs join()

## join()

```java
t1.join();
t2.join();
```

Wait for specific threads.

***

## CountDownLatch

```java
latch.await();
```

Wait for a count to reach zero.

Not tied to specific threads.

# Common Interview Questions

## Does await() release locks?

No.

`await()` blocks the current thread until count becomes zero.

***

## What exception does await() throw?

```java
InterruptedException
```

***

## Can multiple threads call await()?

Yes.

All waiting threads are released when count reaches zero.

***

## Is CountDownLatch thread-safe?

Yes.

It is designed for concurrent use.
