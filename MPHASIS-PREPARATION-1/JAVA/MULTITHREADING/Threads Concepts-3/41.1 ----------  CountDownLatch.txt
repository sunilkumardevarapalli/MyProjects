# CountDownLatch Example (`count = 10`)

## What is CountDownLatch?

`CountDownLatch` is used when one or more threads need to wait until a set of operations completes.

Think of it like a gate.

```text
Count = 10

Worker Threads
     |
countDown()
     |
Count becomes 0
     |
Gate Opens
     |
await() thread continues
```

***

## Important Methods

### countDown()

Decreases count by 1.

```java
latch.countDown();
```

***

### await()

Waits until count becomes 0.

```java
latch.await();
```

***

# Example: 10 Employees Logging In

Main thread should start application only after all 10 employees log in.

```java
import java.util.concurrent.CountDownLatch;

class EmployeeTask implements Runnable {

    private CountDownLatch latch;

    public EmployeeTask(CountDownLatch latch) {
        this.latch = latch;
    }

    @Override
    public void run() {

        System.out.println(
                Thread.currentThread().getName()
                        + " Logged In");

        latch.countDown();

        System.out.println(
                "Remaining Count : "
                        + latch.getCount());
    }
}
```

## Main Class

```java
import java.util.concurrent.CountDownLatch;

public class Test {

    public static void main(String[] args)
            throws InterruptedException {

        CountDownLatch latch =
                new CountDownLatch(10);

        for(int i = 1; i <= 10; i++) {

            Thread t = new Thread(
                    new EmployeeTask(latch));

            t.setName("Employee-" + i);

            t.start();
        }

        System.out.println(
                "Main Thread Waiting...");

        latch.await();

        System.out.println(
                "All Employees Logged In");

        System.out.println(
                "Application Started");
    }
}
```

# Internal Working

Initial Count:

```java
CountDownLatch latch =
        new CountDownLatch(10);
```

```text
Count = 10
```

### Employee-1

```java
latch.countDown();
```

```text
Count = 9
```

### Employee-2

```java
latch.countDown();
```

```text
Count = 8
```

...

### Employee-10

```java
latch.countDown();
```

```text
Count = 0
```

Now:

```java
latch.await();
```

unblocks.

# Timed await()

Instead of waiting forever:

```java
latch.await(
        10,
        TimeUnit.SECONDS);
```

Example:

```java
boolean result =
        latch.await(
                10,
                TimeUnit.SECONDS);

System.out.println(result);
```

Output:

```text
true
```

if count becomes 0 within 10 seconds.

Otherwise:

```text
false
```

# Difference Between join() and CountDownLatch

### join()

```java
t1.join();
t2.join();
t3.join();
```

Wait for specific threads.

***

### CountDownLatch

```java
latch.await();
```

Wait for a count to reach 0.

Not necessarily tied to specific threads.

***

# Interview Answer

`CountDownLatch` is a synchronization utility that allows one or more threads to wait until a set of operations completes. 
It is initialized with a count value. 

Each worker thread calls `countDown()` when its task is finished, decreasing the count. 

Threads calling `await()` remain blocked until the count reaches zero. 

It is commonly used for application startup, batch processing, parallel task completion, and microservice initialization.
