# `join()` Method in Java Threads – Complete Interview Guide

## What is `join()`?

`join()` is used when one thread needs to wait for another thread to complete its execution.

### Definition

> `join()` causes the current thread to wait until the target thread finishes.

***

## Why Do We Need `join()`?

Suppose:

* Thread-1 calculates salary
* Main thread generates report

The report should be generated only after salary calculation completes.

Here we use:

```java
t1.join();
```

***

# Basic Example

```java
class MyThread extends Thread {

    @Override
    public void run() {

        for(int i=1;i<=5;i++) {

            System.out.println(
                    "Child Thread : " + i);
        }
    }
}

public class Test {

    public static void main(String[] args)
            throws Exception {

        MyThread t1 = new MyThread();

        t1.start();

        t1.join();

        System.out.println(
                "Main Thread Completed");
    }
}
```

## Output

```text
Child Thread : 1
Child Thread : 2
Child Thread : 3
Child Thread : 4
Child Thread : 5

Main Thread Completed
```

### What Happened?

```text
Main Thread
      |
      | t1.start()
      |
      | t1.join()
      |
      | WAITING
      |
Thread-1 completes
      |
      V
Main Thread resumes
```

***

```java

```

Internally:

```text

        |
        | waits
        |
        V
      T1
```

Main thread enters:

```text
WAITING State
```

```text
T1 TERMINATED
```

# join(long millis)

Wait only for specific time.

## Example

```java
t1.join(3000);
```

Meaning:

```text
Wait maximum 3 seconds
```

After 3 seconds:

```text
Resume execution
```

even if t1 is still running.

***

## Example

```java
class MyThread extends Thread {

    @Override
    public void run() {

        try {

            Thread.sleep(5000);

        } catch(Exception e) {
        }

        System.out.println(
                "Child Finished");
    }
}
```

Main:

```java
MyThread t1 = new MyThread();

t1.start();

t1.join(2000);

System.out.println(
        "Main Resumed");
```

Output

```text
Main Resumed

Child Finished
```

Because:

```text
join waited only 2 seconds
```

***

# join(long millis, int nanos)

```java
t1.join(5000, 500);
```

Meaning:

```text
Wait 5 seconds and 500 nanoseconds
```

Rarely used.

```java
Thread t1 = new Thread();
```

Output:



# Can a Thread Join Itself?

```java
Thread.currentThread().join();
```

Example:

```java
public void run() {

    Thread.currentThread().join();
}
```

Result:

```text
Deadlock
```

The thread waits for itself forever.

***

# Deadlock Example Using join

```java
public class Test {

    public static void main(String[] args)

        Thread.currentThread().join();

        System.out.println("Hello");
    }
}
```

Output:

```text
Program hangs forever
```

Reason:

```text
Main Thread
waiting for
Main Thread
```

# join() vs sleep()

## sleep()

```java
Thread.sleep(5000);
```

Meaning:

```text
Pause current thread
for 5 seconds
```

***

## join()

```java
t1.join();
```

Meaning:

```text
Wait until thread t1
finishes execution
```

***

## Difference

| Feature                   | sleep()        | join()                  |
| ------------------------- | -------------- | ----------------------- |
| Purpose                   | Pause Thread   | Wait For Another Thread |
| Class                     | Thread         | Thread                  |
| Lock Released             | No             | No                      |
| Depends On Another Thread | No             | Yes                     |
| State                     | TIMED\_WAITING | WAITING/TIMED\_WAITING  |

# Most Asked Interview Questions

### Does join() release lock?

No.

```text
join() does not release locks held by the thread.
```

***

### What state does a thread enter during join()?

```text
WAITING
```

or

```text
TIMED_WAITING
```

for timed join.

***

### Can join() throw exception?

Yes.

```java
InterruptedException
```

***

### Is join() static?

No.

```java
t1.join();
```

is called on a Thread object.

# Interview Answer (1 Minute)

> `join()` is a method of the Thread class used to make one thread wait for another thread to complete.
 When a thread calls `t1.join()`, it enters the WAITING state until thread `t1` finishes execution.
 
 
 Timed versions such as `join(long millis)` allow waiting only for a specific duration. `join()`
 is commonly used when tasks have dependencies, such as report generation after data processing or statement generation after 
 interest calculation. Unlike `sleep()`, which pauses the current thread for a fixed time, `join()` waits for the completion of
 another thread.
