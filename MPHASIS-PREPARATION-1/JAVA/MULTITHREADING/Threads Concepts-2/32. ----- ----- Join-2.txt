# 1. `join()` with Multiple Threads

Suppose we have 3 worker threads:

```java
class WorkerThread extends Thread {

    private String taskName;

    public WorkerThread(String taskName) {
        this.taskName = taskName;
    }

    @Override
    public void run() {

        System.out.println(
                taskName + " Started");

        try {
            Thread.sleep(3000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        System.out.println(
                taskName + " Finished");
    }
}
```

## Main Class

```java
public class Test {

    public static void main(String[] args)
            throws Exception {

        WorkerThread t1 =
                new WorkerThread("Task-1");

        WorkerThread t2 =
                new WorkerThread("Task-2");

        WorkerThread t3 =
                new WorkerThread("Task-3");

        t1.start();
        t2.start();
        t3.start();

        t1.join();
        t2.join();
        t3.join();

        System.out.println(
                "All Tasks Completed");
    }
}
```

### Possible Output

```text
Task-1 Started
Task-2 Started
Task-3 Started

Task-1 Finished
Task-2 Finished
Task-3 Finished

All Tasks Completed
```

### What Happens?

```text
Main Thread
      |
      | start()
      |
      |----> T1
      |----> T2
      |----> T3
      |
      | t1.join()
      | t2.join()
      | t3.join()
      |
      V
Waiting
      |
      V
All threads finish
      |
      V
Main resumes
```

# 2. `join()` with Timed Wait

Java provides:

```java
join(long millis)
```

Instead of waiting forever, wait only for a specific time.

***

## Example

```java
class ReportThread extends Thread {

    @Override
    public void run() {

        try {

            System.out.println(
                    "Report Generation Started");

            Thread.sleep(5000);

            System.out.println(
                    "Report Generation Finished");

        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}
```

## Main

```java
public class Test {

    public static void main(String[] args)
            throws Exception {

        ReportThread t1 =
                new ReportThread();

        t1.start();

        t1.join(2000);

        System.out.println(
                "Main Thread Resumed");
    }
}
```

### Output

```text
Report Generation Started

Main Thread Resumed

Report Generation Finished
```

### Why?

```text
Thread execution time = 5 sec

join timeout = 2 sec
```

Main waits only 2 seconds.

After timeout:

```text
Main thread continues
```

even though child thread is still running.

***

## Timeline

```text
0 sec
 |
 | T1 Started
 |
 | Main calls join(2000)
 |
2 sec
 |
 | Main Resumes
 |
5 sec
 |
 | T1 Finishes
```

***

# Timed Join with Multiple Threads

```java
t1.join(3000);
t2.join(3000);
t3.join(3000);
```

Each join waits a maximum of:

```text
3 seconds
```

for that specific thread.

# 3. What Happens if `join()` Causes Deadlock?

A deadlock can happen if a thread waits for itself.

***

## Example 1: Self Join

```java
public class Test {

    public static void main(String[] args)
            throws Exception {

        Thread.currentThread().join();

        System.out.println("Hello");
    }
}
```

### Output

```text
Program hangs forever
```

### Why?

```text
Main Thread
    |
    | waiting for
    |
Main Thread
```

It can never complete.

***

## Diagram

```text
Main Thread
     |
     V
Waiting For
     |
     V
Main Thread
```

Deadlock.

***

# Example 2: Mutual Join Deadlock

```java
class MyThread extends Thread {

    Thread other;

    public void setOther(Thread other) {
        this.other = other;
    }

    @Override
    public void run() {

        try {

            other.join();

        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        System.out.println(
                Thread.currentThread().getName()
                        + " Finished");
    }
}
```

## Main

```java
public class Test {

    public static void main(String[] args) {

        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();

        t1.setOther(t2);
        t2.setOther(t1);

        t1.start();
        t2.start();
    }
}
```

### What Happens?

```text
T1 waiting for T2

T2 waiting for T1
```

***

## Diagram

```text
T1
 |
 | waiting for
 v
T2

T2
 |
 | waiting for
 v
T1
```

Result:

```text
Deadlock
```

Neither thread finishes.

***

# How to Avoid `join()` Deadlocks

### ✅ Never join the current thread

Wrong:

```java
Thread.currentThread().join();
```

***

### ✅ Avoid circular dependencies

Wrong:

```text
T1 waits for T2
T2 waits for T1
```

***

### ✅ Use proper hierarchy

Example:

```text
Main waits for T1
Main waits for T2
Main waits for T3
```

Not:

```text
T1 waits for T2
T2 waits for T1
```

***

# `join()` vs `sleep()`

### sleep()

```java
Thread.sleep(5000);
```

Meaning:

```text
Pause current thread for 5 seconds
```

***

### join()

```java
t1.join();
```

Meaning:

```text
Wait until t1 completes
```

***

# Interview Answer

> `join()` makes the current thread wait for another thread to complete. With multiple threads, the main thread can call `join()` 
on each worker thread to ensure all tasks finish before continuing. Timed joins such as `join(2000)` 
wait only for the specified duration and then resume even if the target thread is still running.
 Deadlock can occur if a thread calls `join()` on itself or if two threads wait for each other using `join()`. 
 To avoid deadlocks, never create circular waiting dependencies between threads.
