# Transient Variable in Java (Complete Interview Guide)

`transient` is a keyword used in Java Serialization.

## What is a Transient Variable?

A variable declared with the `transient` keyword will **not be serialized**.

```java
transient String password;
```

### Definition

> Transient variables are ignored during serialization and are not saved into the object stream.

***

# What is Serialization?

Serialization means converting an object into a byte stream so that it can be:

* Stored in a file
* Sent over a network
* Saved in a database
* Transferred between JVMs

Example:

```java
ObjectOutputStream oos =
        new ObjectOutputStream(
                new FileOutputStream("emp.ser"));

oos.writeObject(employee);
```

***

# What is Deserialization?

Converting byte stream back into an object.

```java
ObjectInputStream ois =
        new ObjectInputStream(
                new FileInputStream("emp.ser"));

Employee emp =
        (Employee) ois.readObject();
```

***

# Why Do We Need Transient?

Some data should not be stored.

Examples:

* Passwords
* OTPs
* Session IDs
* Security Tokens
* Temporary Data
* Cache Data

Example:

```java
class Customer {

    String name;

    transient String password;
}
```

When serialized:

```text
name      -> Saved
password  -> Not Saved
```

***

# Serialization Example

## Employee Class

```java
import java.io.Serializable;

public class Employee
        implements Serializable {

    private int id;
    private String name;

    private transient String password;

    public Employee(int id,
                    String name,
                    String password) {

        this.id = id;
        this.name = name;
        this.password = password;
    }

    @Override
    public String toString() {

        return id + " "
                + name + " "
                + password;
    }
}
```

***

## Serialize Object

```java
import java.io.*;

public class SerializeDemo {

    public static void main(String[] args)
            throws Exception {

        Employee emp =
                new Employee(
                        101,
                        "Sunil",
                        "Admin123");

        ObjectOutputStream oos =
                new ObjectOutputStream(
                        new FileOutputStream("emp.ser"));

        oos.writeObject(emp);

        oos.close();

        System.out.println(
                "Object Serialized");
    }
}
```

***

## Deserialize Object

```java
import java.io.*;

public class DeserializeDemo {

    public static void main(String[] args)
            throws Exception {

        ObjectInputStream ois =
                new ObjectInputStream(
                        new FileInputStream("emp.ser"));

        Employee emp =
                (Employee) ois.readObject();

        System.out.println(emp);

        ois.close();
    }
}
```

### Output

```text
101 Sunil null
```

Notice:

```text
password = null
```

because:

```java
transient String password;
```

was skipped during serialization.

***

# What Value Does a Transient Variable Get?

After deserialization, transient variables get default values.

## String

```java
transient String name;
```

Value:

```text
null
```

***

## int

```java

```

Value:

```text

```

***

## boolean

```java

```

Value:

```text
false
```

***

## double

```java
transient double salary;
```

Value:

```text
0.0
```

# Can Static Variables Be Serialized?

```java
static int count;
```

No.

Static variables belong to the class, not the object.

Therefore:

```text
Serialization ignores static variables.
```

Example:

```java
static String companyName;
```

Not serialized.

***

# Static vs Transient

```java
transient String password;
```

Not serialized because of:

```text
transient
```

***

```java
static String company;
```

Not serialized because:

```text
static belongs to class
```

***

# Can We Use Both Together?

```java
transient static int count;
```

Valid syntax.

But:

```text
Static variable is already ignored.
```

So transient becomes meaningless here.

# Transient and Inheritance

```java

        implements Serializable {
```

After deserialization:

```text

```

because transient fields are not serialized.

# Important Interview Scenario

```java
class Employee
        implements Serializable {

    int id = 100;

    transient int salary = 50000;

```

Before Serialization:

```text
id = 100
salary = 50000
```

After Deserialization:

```text
id = 100
salary = 0
```

Why?

Because:

```text
salary was not stored
```

# Difference Between Static and Transient

| Feature                   | static               | transient          |
| ------------------------- | -------------------- | ------------------ |
| Belongs To                | Class                | Object             |
| Serialized                | No                   | No                 |
| Purpose                   | Shared Data          | Skip Serialization |
| Memory                    | Class Area           | Object Memory      |
| Default After Deserialize | Current Static Value | Default Value      |

# Most Asked Interview Questions

### Why use transient?

To prevent sensitive or unnecessary fields from being serialized.

***

### Will transient variable be saved to file?

No.

***

### What value will transient variable have after deserialization?

Default value.

Example:

```text
int -> 0
boolean -> false
String -> null
```

***

### Can transient be applied to static variables?

Yes syntactically, but it is unnecessary because static variables are already not serialized.

***

### Does transient improve security?

Yes.

Sensitive data such as passwords, OTPs, and tokens are not stored in serialized objects.

***

# Interview Answer (2 Minutes)

> The `transient` keyword is used in Java Serialization to exclude a variable from the serialization process. 
When an object is serialized, transient fields are not written to the object stream. During deserialization,
 these fields receive their default values such as `null`, `0`, or `false`. 
 Transient is commonly used for sensitive information like passwords, OTPs, session tokens, authentication details,
 and temporary cache data. It helps improve security and reduces unnecessary data storage. Static variables are also not
 serialized, but for a different reason—they belong to the class rather than the object.
