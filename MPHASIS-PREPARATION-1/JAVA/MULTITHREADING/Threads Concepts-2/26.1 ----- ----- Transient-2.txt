## 1. How Does `transient` Affect Inheritance?

`transient` works at the **field level**, not at the inheritance level.

If a parent class has a transient field and that field is serialized through a child object, the transient field is still skipped.

### Example

```java
import java.io.Serializable;

class Parent implements Serializable {

    transient int salary = 50000;

    String name = "Sunil";
}

class Child extends Parent {

    int id = 101;
}
```

Before Serialization:

```text
salary = 50000
name   = Sunil
id     = 101
```

After Deserialization:

```text
salary = 0
name   = Sunil
id     = 101
```

Explanation:

* `salary` is transient → not serialized
* `name` is serialized
* `id` is serialized

### Interview Point

> Transient fields are not inherited as serialized data. Even if the child object is serialized, transient fields from the parent class are skipped and restored with default values during deserialization.

***

## If Parent is Not Serializable

### Example

```java
class Parent {

    int salary = 50000;
}

class Child extends Parent implements Serializable {

    int id = 101;
}
```

When deserializing:

* Parent constructor executes
* Parent fields get initialized by constructor
* Child serialized fields are restored from stream

This behavior is due to serialization rules, not the transient keyword itself.

***

# 2. Can `transient` Be Used With `final`?

### Yes

This is perfectly valid:

```java
transient final int age = 25;
```

or

```java
final transient int age = 25;
```

Both compile.

### Example

```java
class Employee implements Serializable {

    transient final int age = 25;
}
```

### After Deserialization

```text
age = 25
```

Why?

Because:

```java
final int age = 25;
```

is initialized during object construction.

The value comes from initialization, not from the serialized stream.

***

### Another Example

```java
class Employee implements Serializable {

    transient final String company =
            "TCS";
}
```

After Deserialization:

```text
company = TCS
```

Still available.

***

### Interview Question

Can `transient final` be used?

**Answer:** Yes.

However, it is uncommon because final fields are usually constants or initialized values and often don't need to be excluded 
from serialization.

***

# 3. What Happens If `transient` Is Omitted?

This is the most important interview question.

Suppose:

```java
class Customer implements Serializable {

    String username;

    String password;
}
```

### Object

```java
Customer c = new Customer();

c.username = "Sunil";
c.password = "Admin123";
```

### Serialization

Both values are stored:

```text
username = Sunil
password = Admin123
```

### Deserialization

```text
username = Sunil
password = Admin123
```

Everything is restored.

***

## Using `transient`

```java
class Customer implements Serializable {

    String username;

    transient String password;
}
```

### Serialization

Stored:

```text
username = Sunil
```

Skipped:

```text
password
```

### Deserialization

```text
username = Sunil
password = null
```

***

# Real-Time Problem If `transient` Is Omitted

### Banking Application

```java
class UserAccount
        implements Serializable {

    String userName;

    String password;

    String otp;
}
```

Without transient:

```text
Password saved to file
OTP saved to file
```

Security risk ❌

***

### Correct Solution

```java
class UserAccount
        implements Serializable {

    String userName;

    transient String password;

    transient String otp;
}
```

Now:

```text
Username saved
Password skipped
OTP skipped
```

Safe ✅

***

# Interview Summary

### How does transient affect inheritance?

> Transient fields remain non-serializable even when inherited. If a child object is serialized, inherited transient fields are skipped and restored with default values during deserialization.

### Can transient be used with final?

> Yes. `transient final` is valid. After deserialization, final fields generally retain their initialized values because they are initialized during object creation rather than restored from the serialized stream.

### What happens if transient is omitted?

> The field becomes part of the serialized object. Its value is written to the stream and restored during deserialization. Omitting transient for sensitive data such as passwords, OTPs, tokens, or session information can create security risks.
