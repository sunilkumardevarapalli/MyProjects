# When to Avoid Using `volatile`?

Use `volatile` only for **visibility**. Avoid it when you need **thread safety** or **atomicity**.

## ❌ Avoid for Increment/Decrement

```java
class Counter {

    volatile int count = 0;

    public void increment() {
        count++;
    }
}
```

Problem:

```text
count++

Internally:

Read
Modify
Write
```

Multiple threads can overwrite each other's updates.

Use:

```java
AtomicInteger
```

or

```java
synchronized
```

***

## ❌ Avoid for Bank Transactions

```java
volatile double balance = 10000;

balance = balance - 500;
```

Problems:

```text
ATM Thread
GooglePay Thread
PhonePe Thread
```

All may update the same balance simultaneously.

Use:

```java
synchronized
```

***

## ❌ Avoid for Collections

```java
volatile List<String> customers =
        new ArrayList<>();
```

Problem:

```java
customers.add("Sunil");
```

`volatile` makes only the reference visible.

It does NOT make:

```java
add()
remove()
clear()
```

thread-safe.

Use:

```java
CopyOnWriteArrayList
ConcurrentHashMap
```

***

## ❌ Avoid for Check-Then-Act Logic

```java
if(balance > 1000) {
    balance -= 1000;
}
```

Multiple threads may pass the check simultaneously.

Use:

```java
synchronized
```

***

## Use `volatile` For

```java
volatile boolean shutdown;

volatile boolean serverRunning;

volatile boolean cacheEnabled;

volatile String currentVersion;
```

***

# When to Avoid Using `transient`?

Avoid `transient` when data must be preserved after serialization.

***

## ❌ Customer Information

```java
class Customer
        implements Serializable {

    transient String customerName;
}
```

After deserialization:

```text
customerName = null
```

You lose important business data.

***

## ❌ Employee Data

```java
transient int employeeId;
```

After deserialization:

```text
employeeId = 0
```

Not desirable.

***

## ❌ Database Entity Fields

```java
transient Long orderId;
```

Order details may be lost after serialization.

***

## ❌ Audit Information

```java
transient String auditStatus;
```

Audit information is usually important and should be saved.

***

## Use `transient` For

```java
transient String password;

transient String otp;

transient String sessionToken;

transient String captcha;
```

***

# Complete Example Using static, transient and volatile

```java
import java.io.Serializable;

public class Employee
        implements Serializable {

    private int id;

    private String name;

    // Shared by all objects
    static String companyName = "Microsoft";

    // Not serialized
    transient String password;

    // Visible across threads
    volatile boolean active;

    public Employee(int id,
                    String name,
                    String password,
                    boolean active) {

        this.id = id;
        this.name = name;
        this.password = password;
        this.active = active;
    }

    @Override
    public String toString() {

        return "Employee{" +
                "id=" + id +
                ", name='" + name + '\'' +
                ", password='" + password + '\'' +
                ", active=" + active +
                ", companyName='" + companyName + '\'' +
                '}';
    }
}
```

# Behavior During Serialization

```java
Employee emp =
        new Employee(
                101,
                "Sunil",
                "Admin@123",
                true);
```

Before Serialization:

```text
id=101
name=Sunil
password=Admin@123
active=true
companyName=Microsoft
```

After Deserialization:

```text
id=101
name=Sunil
password=null
active=true
companyName=current static value
```

Explanation:

### id

```text
Serialized ✅
```

### name

```text
Serialized ✅
```

### password

```text
Transient
Not Serialized ❌
```

Result:

```text
null
```

### active

```text
Volatile does NOT affect serialization
Serialized ✅
```

### companyName

```text
Static
Not Serialized ❌
```

JVM uses the current class value.

# One-Line Interview Answer

* **volatile** → Ensures changes made by one thread are immediately visible to other threads.
* **transient** → Excludes a field from serialization.
* **static** → Variable belongs to the class and is shared among all objects.
