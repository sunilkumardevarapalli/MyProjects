# 1. Circular Join Deadlock

A **circular join deadlock** happens when two threads wait for each other to finish using `join()`.

## Example

```java
class MyThread extends Thread {

    private Thread otherThread;

    public void setOtherThread(Thread otherThread) {
        this.otherThread = otherThread;
    }

    @Override
    public void run() {

        try {

            System.out.println(
                    getName() + " waiting for "
                            + otherThread.getName());

            otherThread.join();

        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        System.out.println(getName() + " completed");
    }
}
```

```java
public class Main {

    public static void main(String[] args) {

        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();

        t1.setName("T1");
        t2.setName("T2");

        t1.setOtherThread(t2);
        t2.setOtherThread(t1);

        t1.start();
        t2.start();
    }
}
```

## Output

```text
T1 waiting for T2
T2 waiting for T1
```

Program hangs forever.

## Diagram

```text
T1
 |
 | join()
 v
T2

T2
 |
 | join()
 v
T1
```

Both threads are waiting forever.

This is called a **circular wait**, one of the major conditions for deadlock.

# 2. How to Prevent Self-Join Deadlock?

## Wrong Code

```java
public class Test {

    public static void main(String[] args)
            throws Exception {

        Thread.currentThread().join();

        System.out.println("Hello");
    }
}
```

## What Happens?

```text
Main Thread
  |
  | waiting for
  |
Main Thread
```

The thread is waiting for itself to complete.

This can never happen.

Program hangs forever.

***

## Prevention

Before calling join, ensure:

```java
if(Thread.currentThread() != targetThread) {

    targetThread.join();
}
```

### Example

```java
Thread t1 = new Thread();

if(Thread.currentThread() != t1) {

    t1.join();
}
```

***

## Best Practice

Only the parent/coordinator thread should call join.

Example:

```java
t1.start();
t2.start();
t3.start();

t1.join();
t2.join();
t3.join();
```

Avoid:

```text
T1 waits for T2
T2 waits for T1
```

# 3. Difference Between join() and sleep()

This is one of the most asked interview questions.

## join()

```java
t1.join();
```

Meaning:

```text
Wait until thread t1 finishes.
```

### Example

```java
t1.start();

t1.join();

System.out.println("Main Continues");
```

Output:

```text
T1 Finished

Main Continues
```

The current thread waits for another thread.

***

## sleep()

```java
Thread.sleep(5000);
```

Meaning:

```text
Pause current thread for 5 seconds.
```

### Example

```java
System.out.println("Start");

Thread.sleep(5000);

System.out.println("End");
```

Output:

```text
Start

(wait 5 seconds)

End
```

No dependency on another thread.

***

# Example Comparison

## Using sleep()

```java
Thread t1 = new Thread(() -> {

    try {
        Thread.sleep(3000);
    } catch (Exception e) {
    }

    System.out.println("Child Finished");
});

t1.start();

Thread.sleep(1000);

System.out.println("Main Finished");
```

Possible Output

```text
Main Finished
Child Finished
```

Main waited only 1 second.

***

## Using join()

```java
Thread t1 = new Thread(() -> {

    try {
        Thread.sleep(3000);
    } catch (Exception e) {
    }

    System.out.println("Child Finished");
});

t1.start();

t1.join();

System.out.println("Main Finished");
```

Output

```text
Child Finished
Main Finished
```

Main waits for child completion.

***

# join() vs sleep()

| Feature            | join()                              | sleep()              |
| ------------------ | ----------------------------------- | -------------------- |
| Purpose            | Wait for another thread             | Pause current thread |
| Dependency         | Depends on target thread completion | Depends on time      |
| Method Call        | `t1.join()`                         | `Thread.sleep()`     |
| Waiting State      | WAITING / TIMED\_WAITING            | TIMED\_WAITING       |
| Used For           | Task dependency                     | Delay                |
| Can Cause Deadlock | Yes                                 | No                   |

***

# Interview Answer

> `join()` is used when one thread needs to wait for another thread to finish execution, whereas `sleep()` 
simply pauses the current thread for a specified amount of time. A circular join deadlock occurs when two threads call
 `join()` on each other and wait forever. A self-join deadlock occurs when a thread calls `join()` on itself. 
 These deadlocks can be avoided by preventing circular dependencies and ensuring a thread never waits on itself.
