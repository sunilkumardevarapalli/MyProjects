## Object-Level Lock (Instance-Level Lock) in Java

When a synchronized **instance method** is called, Java acquires a lock on the **object (instance)**, not on the class.

**Important Rule:**

* One object = One lock
* Different objects = Different locks

So, if two threads are using **different objects**, they can execute synchronized methods simultaneously.

***

### Example 1: Same Object

```java
class Employee {

    synchronized void method1() {
        System.out.println("method1 started");
        try { Thread.sleep(5000); } catch (Exception e) {}
        System.out.println("method1 finished");
    }

    synchronized void method2() {
        System.out.println("method2 started");
    }
}
```

```java
Employee emp = new Employee();

Thread t1 = new Thread(() -> emp.method1());
Thread t2 = new Thread(() -> emp.method2());

t1.start();
t2.start();
```

### What happens?

Thread-1 enters `method1()`.

Java acquires lock on `emp`.

```text
emp LOCK = occupied by Thread-1
```

Now Thread-2 tries to enter `method2()`.

```text
Thread-2 waits
```

Because both methods are synchronized and both need the same object's lock.

Result:

```text
Thread-1 gets permission ✅
Thread-2 waits ❌

After Thread-1 exits:

Thread-2 gets permission ✅
```

***

## Example 2: Two Different Objects

```java
Employee emp1 = new Employee();
Employee emp2 = new Employee();

Thread t1 = new Thread(() -> emp1.method1());
Thread t2 = new Thread(() -> emp2.method2());

t1.start();
t2.start();
```

Here:

```text
emp1 has its own lock
emp2 has its own lock
```

### Execution

Thread-1:

```text
locks emp1
executes method1
```

Thread-2:

```text
locks emp2
executes method2
```

Both run simultaneously.

Result:

```text
Thread-1 gets permission ✅
Thread-2 gets permission ✅
```

Because locks are different.

***

## Interview Scenario

Consider:

```java
class Test {

    synchronized void m1() {
    }

    synchronized void m2() {
    }
}
```

And:

```java
Test obj1 = new Test();
Test obj2 = new Test();
```

### Case 1

```java
Thread-1 --> obj1.m1()
Thread-2 --> obj1.m2()
```

Same object.

```text
obj1 lock required by both methods
```

Result:

```text
Only one thread runs at a time.
```

***

### Case 2

```java
Thread-1 --> obj1.m1()
Thread-2 --> obj2.m2()
```

Different objects.

```text
obj1 lock
obj2 lock
```

Result:

```text
Both threads execute concurrently.
```

***

## Visual Representation

### Same Object

```text
                |

Result:

```

### Different Objects

```text
Thread-1 ---> obj1.m1()
                |
                | LOCK obj1

                |

Result:
Both execute simultaneously
```

## Most Asked Interview Question

### Does a synchronized method lock the method?

**No.**

A very common misconception.

Java never locks the method.

Java locks the **object** on which the synchronized method is called.

Example:

```java
obj1.m1();
```

Java acquires:

```text
Lock on obj1
```

Not:

```text
Lock on m1()
```

***

## 30-Second Interview Answer

> Object-level (instance-level) lock means the lock is associated with a specific object instance. 
When a thread enters a synchronized instance method, it acquires that object's lock. If another thread tries to access any synchronized method of the same object, it must wait. However, if the threads use different objects, they acquire different locks and can execute synchronized methods simultaneously.
