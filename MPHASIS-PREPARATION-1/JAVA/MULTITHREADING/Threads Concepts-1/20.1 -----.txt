# 1. How to Handle Exceptions in Scheduled Tasks?

This is a very important interview question.

## Problem

Suppose your scheduled task throws an exception.

```java
ScheduledExecutorService ex =
        Executors.newScheduledThreadPool(2);

ex.scheduleAtFixedRate(() -> {

    System.out.println("Running Task");

    int x = 10 / 0;

}, 0, 5, TimeUnit.SECONDS);
```

### Output

```text
Running Task

Exception...
```

After exception:

```text
Task Stops Permanently
```

Future executions will not happen.

***

## Best Practice: Handle Exception Inside Runnable

```java
public class TriggerEmailPolicy implements Runnable {

    @Override
    public void run() {

        try {

            System.out.println(
                    Thread.currentThread().getName()
                    + " Sending Email");

            int x = 10 / 0;

        } catch (Exception e) {

            System.out.println(
                    "Error : "
                    + e.getMessage());
        }
    }
}
```

### Result

```text
Execution-1

Error : / by zero

Execution-2

Error : / by zero

Execution-3

Error : / by zero
```

Task keeps running.

***

## Handling Exceptions With ScheduledFuture

```java
ScheduledFuture<?> future =
        ex.scheduleAtFixedRate(task,
                0,
                5,
                TimeUnit.SECONDS);
```

Later:

```java
try {

    future.get();

} catch (ExecutionException e) {

    System.out.println(
            e.getCause());

}
```

Useful when you need to know why the task failed.

# 2. Can I Use scheduleWithFixedDelay Instead?

### Yes.

Java provides:

```java
scheduleAtFixedRate()
```

and

```java
scheduleWithFixedDelay()
```

Both are frequently asked in interviews.

***

# scheduleAtFixedRate()

Syntax

```java
ex.scheduleAtFixedRate(
        task,
        initialDelay,
        period,
        TimeUnit.SECONDS);
```

Example

```java
ex.scheduleAtFixedRate(
        task,
        0,
        10,
        TimeUnit.SECONDS);
```

Meaning:

```text
Start every 10 seconds
regardless of previous execution completion
```

Timeline

```text
Start
  |
0 sec
  |
10 sec
  |
20 sec
  |
30 sec
```

***

## Example

Task Time:

```text
3 seconds
```

Period:

```text
10 seconds
```

Execution:

```text
0 sec  -> Start

3 sec  -> Finish

10 sec -> Start Again

13 sec -> Finish

20 sec -> Start Again
```

***

# scheduleWithFixedDelay()

Syntax

```java
ex.scheduleWithFixedDelay(
        task,
        initialDelay,
        delay,
        TimeUnit.SECONDS);
```

Example

```java
ex.scheduleWithFixedDelay(
        task,
        0,
        10,
        TimeUnit.SECONDS);
```

Meaning:

```text
Wait 10 seconds AFTER previous execution finishes
```

***

## Example

Task Duration:

```text
3 seconds
```

Delay:

```text
10 seconds
```

Execution:

```text
0 sec -> Start

3 sec -> Finish

13 sec -> Start Again

16 sec -> Finish

26 sec -> Start Again
```

***

# Difference

## scheduleAtFixedRate

```text
Start-to-Start
```

Fixed schedule.

```text
0
10
20
30
40
```

Use Cases:

* Health Checks
* Monitoring
* Metrics Collection
* Heartbeat Jobs

***

## scheduleWithFixedDelay

```text
Finish + Delay
```

Use Cases:

* Email Processing
* File Processing
* Batch Jobs
* Data Synchronization

***

# Interview Answer

### scheduleAtFixedRate

```text
Next execution time
=
Previous scheduled start time
+
Period
```

### scheduleWithFixedDelay

```text
Next execution time
=
Previous finish time
+
Delay
```

# 3. How to Schedule Tasks With Dynamic Intervals?

Suppose:

```text
1st Run -> after 5 sec

2nd Run -> after 10 sec

3rd Run -> after 20 sec

4th Run -> after 40 sec
```

This cannot be achieved directly using:

```java
scheduleAtFixedRate()
```

or

```java
scheduleWithFixedDelay()
```

because they use fixed intervals.

***

## Dynamic Scheduling Example

```java
import java.util.concurrent.*;

public class DynamicScheduler {

    private static final ScheduledExecutorService ex =
            Executors.newScheduledThreadPool(1);

    private static int delay = 5;

    public static void main(String[] args) {

        scheduleTask();
    }

    private static void scheduleTask() {

        ex.schedule(() -> {

            System.out.println(
                    "Executing with delay : "
                    + delay);

            delay = delay * 2;

            scheduleTask();

        }, delay, TimeUnit.SECONDS);
    }
}
```

***

### Output

```text
Executing with delay : 5

Executing with delay : 10

Executing with delay : 20

Executing with delay : 40

Executing with delay : 80
```

***

# Real-Time Dynamic Scheduling Examples

### Retry Mechanism

```text
Try-1 -> 5 sec

Try-2 -> 10 sec

Try-3 -> 20 sec

Try-4 -> 40 sec
```

Called:

```text
Exponential Backoff
```

Used in:

* Payment Gateway Retries
* Kafka Consumers
* Microservices
* API Calls

***

### Email Retry

```text
Email Failed

Retry after 1 minute

Retry after 5 minutes

Retry after 15 minutes
```

Dynamic delays.

***

### Cache Refresh

```text
Normal Load -> Every 30 min

High Load -> Every 5 min
```

Dynamic interval.

# Most Asked Interview Comparison

```java
schedule()
```

Runs once.

***

```java
scheduleAtFixedRate()
```

Runs repeatedly at fixed intervals.

***

```java
scheduleWithFixedDelay()
```

Runs repeatedly after waiting a fixed delay from previous completion.

***

```java
dynamic schedule()
```

Delay changes at runtime.

***

# Interview Summary

* Always handle exceptions inside scheduled tasks; otherwise periodic execution may stop.
* Use `scheduleAtFixedRate()` when you need execution at fixed time intervals.
* Use `scheduleWithFixedDelay()` when the next execution should start after the previous execution finishes plus a delay.
* For changing intervals (retry logic, exponential backoff, dynamic polling), use `schedule()` recursively and calculate the next delay programmatically.
