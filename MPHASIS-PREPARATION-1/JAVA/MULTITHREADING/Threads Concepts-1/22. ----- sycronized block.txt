# Synchronized Block in Java (Object Lock and Class Lock)

A synchronized block is used when you want to synchronize only a specific piece of code instead of the entire method.

## Why Synchronized Block?

Instead of:

```java
public synchronized void withdraw() {

    // 100 lines of code

}
```

You can synchronize only the critical section:

```java
public void withdraw() {

    // non-critical code

    synchronized(this) {

        // critical section
    }

    // non-critical code
}
```

Benefits:

* Better performance
* Smaller lock scope
* Higher concurrency

***

# 1. Object-Level Lock Using Synchronized Block

## Syntax

```java
synchronized(objectReference) {

    // critical section
}
```

or

```java
synchronized(this) {

    // critical section
}
```

Lock acquired on:

```text
Current Object
```

***

## Example

```java
class HdfcBank {

    private int balance = 10000;

    public void withdraw(int amount) {

        synchronized(this) {

            System.out.println(
                    Thread.currentThread().getName()
                    + " entered withdraw");

            balance = balance - amount;

            System.out.println(
                    "Balance : " + balance);

            try {
                Thread.sleep(5000);
            } catch(Exception e) {
            }

            System.out.println(
                    Thread.currentThread().getName()
                    + " leaving withdraw");
        }
    }
}
```

***

## Same Object Scenario

```java
HdfcBank bank = new HdfcBank();

Thread t1 =
    new Thread(() -> bank.withdraw(1000));

Thread t2 =
    new Thread(() -> bank.withdraw(2000));

t1.start();
t2.start();
```

### Execution

```text
Thread-1 acquires bank lock

Thread-2 waits
```

Result:

```text
Only one thread can enter synchronized block
```

***

## Different Objects Scenario

```java
HdfcBank bank1 = new HdfcBank();
HdfcBank bank2 = new HdfcBank();

Thread t1 =
    new Thread(() -> bank1.withdraw(1000));

Thread t2 =
    new Thread(() -> bank2.withdraw(2000));

t1.start();
t2.start();
```

### Execution

```text
Thread-1 --> bank1 lock

Thread-2 --> bank2 lock
```

Result:

```text
Both execute simultaneously
```

Because:

```text
Different Objects
Different Locks
```

***

# Object Lock Example With Multiple Methods

```java
class HdfcBank {

    public void withdraw() {

        synchronized(this) {

            System.out.println("Withdraw");
        }
    }

    public void deposit() {

        synchronized(this) {

            System.out.println("Deposit");
        }
    }
}
```

Same object:

```java
bank.withdraw();
bank.deposit();
```

Both methods need:

```text
bank object's lock
```

Only one thread can execute at a time.

***

# 2. Class-Level Lock Using Synchronized Block

## Syntax

```java
synchronized(ClassName.class) {

}
```

Lock acquired on:

```text
Class Object
```

Example:

```java
synchronized(HdfcBank.class) {

}
```

***

## Example

```java
class HdfcBank {

    private static int balance = 10000;

    public void withdraw(int amount) {

        synchronized(HdfcBank.class) {

            balance = balance - amount;

            System.out.println(
                    Thread.currentThread().getName()
                    + " Balance : "
                    + balance);

            try {
                Thread.sleep(5000);
            } catch(Exception e) {
            }
        }
    }
}
```

***

## Two Objects Scenario

```java
HdfcBank bank1 = new HdfcBank();
HdfcBank bank2 = new HdfcBank();

Thread t1 =
     new Thread(() -> bank1.withdraw(1000));

Thread t2 =
     new Thread(() -> bank2.withdraw(2000));

t1.start();
t2.start();
```

### Lock

```text
HdfcBank.class
```

Both objects use:

```text
Same Lock
```

Result:

```text
Thread-1 enters

Thread-2 waits
```

***

# Visual Representation

## Object Lock

```text
bank1 ---- Lock1
bank2 ---- Lock2
```

Execution:

```text
T1 -> bank1
T2 -> bank2

Allowed Together ✅
```

***

## Class Lock

```text
HdfcBank.class
```

One lock for entire class.

Execution:

```text
T1 -> bank1
T2 -> bank2
```

Result:

```text
Only One Thread Allowed ✅
```

# Object Lock vs Class Lock

### Object Lock

```java
synchronized(this)
```

or

```java
synchronized(object)
```

Lock:

```text
Object Instance
```

Objects:

```text
obj1 lock
obj2 lock
obj3 lock
```

Every object gets separate lock.

***

### Class Lock

```java
synchronized(HdfcBank.class)
```

Lock:

```text
Class Object
```

Objects:

```text
obj1
obj2
obj3
```

All use same:

```text
HdfcBank.class lock
```

***

# Mixed Scenario (Very Important Interview Question)

```java
class HdfcBank {

    public synchronized void withdraw() {

    }

    public static synchronized void deposit() {

    }
}
```

Thread 1:

```java
bank.withdraw();
```

Needs:

```text
bank lock
```

Thread 2:

```java
HdfcBank.deposit();
```

Needs:

```text
HdfcBank.class lock
```

### Result

```text
Both execute simultaneously
```

Reason:

```text
Different Locks
```

# Common Interview Questions

## Q1: Does synchronized lock method or object?

Answer:

```text
Object
```

Java never locks methods.

It locks:

```text
Object Monitor
```

or

```text
Class Monitor
```

***

## Q2: Can two threads execute two synchronized methods of same object?

```java
obj.m1();
obj.m2();
```

Answer:

```text
No
```

Because both need same object lock.

***

## Q3: Can two threads execute synchronized methods on different objects?

```java
obj1.m1();
obj2.m2();
```

Answer:

```text
Yes
```

Different locks.

***

## Q4: Can two threads execute static synchronized methods on different objects?

```java
obj1.m1();
obj2.m2();
```

Answer:

```text
No
```

Both need:

```text
ClassName.class lock
```

***

# Easy Interview Formula

```text
synchronized(this)
        ↓
Object Lock
```

```text
synchronized(obj)
        ↓
Specified Object Lock
```

```text
synchronized(ClassName.class)
        ↓
Class Lock
```

```text
synchronized instance method
        ↓
Equivalent to synchronized(this)
```

```text
static synchronized method
        ↓
Equivalent to synchronized(ClassName.class)
```

## 1-Minute Interview Answer

> A synchronized block is used to protect a critical section of code. When we use `synchronized(this)` or `synchronized(obj)`, 
Java acquires an object-level lock, allowing only one thread at a time to execute that block for that particular object. 
Different objects have different locks, so they can run concurrently. When we use `synchronized(ClassName.class)`, 
Java acquires a class-level lock. Since only one Class object exists in the JVM, all objects of that class share the same lock, 
and only one thread can execute the synchronized block at a time across all instances.
