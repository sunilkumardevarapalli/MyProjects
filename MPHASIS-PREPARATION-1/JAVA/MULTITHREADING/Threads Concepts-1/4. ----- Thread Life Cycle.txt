## Java Thread Life Cycle

A thread in Java goes through different states during its execution.

### Thread Life Cycle Diagram

	
	NEW
	 |
	 | start()
	 v
	RUNNABLE
	 |
	 | CPU Scheduler
	 v
	RUNNING
	 |
	 | sleep(), wait(), join(), I/O
	 v
	WAITING / TIMED_WAITING / BLOCKED
	 |
	 | notify(), timeout, lock available
	 v
	RUNNABLE
	 |
	 | run() completes
	 v
	TERMINATED
	



## 1. NEW State

	A thread is created but not started.


	Thread t1 = new Thread();


	At this point:


	State = NEW


	### Methods Used

	* `new Thread()`

Once start method will invoke then all the threads are in runnable pool. The OS will call some threads to execute the run method.

When `start()` is called, the thread moves to the Runnable state.

## Runnable State vs Running State

### Runnable State

When you call:

```java
Thread t = new Thread();
t.start();
```

The thread moves to the **RUNNABLE** state.

**Meaning:**

* The thread is ready to run.
* It is waiting for CPU time.
* It sits in the **Ready Queue (Runnable Pool)**.
* The OS/JVM scheduler has not yet assigned a CPU core to execute it.

```text
Runnable Pool

Thread-1
Thread-2
Thread-3
Thread-4
```

All these threads are ready, but only one may get the CPU immediately.

***

### Running State

A thread enters the **RUNNING** state when the CPU scheduler selects it and gives it a CPU core.

```java
public void run() {
    System.out.println("Running...");
}
```

Now instructions inside `run()` are actually executing.

```text
CPU Core
   |
   v
Thread-2 (RUNNING)
```

The remaining threads stay in the Runnable Pool.

***

## Simple Analogy

Imagine you're waiting at a bank.

### Runnable Pool

```text
Customer A
Customer B
Customer C
Customer D
```

Everyone has a token and is waiting.

### Running State

```text
Cashier is serving Customer B
```

Only Customer B is being served.

* Waiting customers = Runnable
* Customer being served = Running

## How Does the OS Give Permission to Run?

A common misunderstanding is:

> "Does Java decide which thread runs?"

Not completely.

### JVM Creates Native Threads

When you write:

```java
t.start();
```

JVM asks the Operating System to create a native thread.

```text
Java Thread
     |
     v
JVM
     |
     v
OS Thread
```

Modern JVMs use **1:1 thread mapping**, meaning one Java thread corresponds to one OS thread.

***

## What Happens Internally?

### Step 1: Thread Created

```java
Thread t1 = new Thread();
```

```text
State = NEW
```

***

### Step 2: start()

```java
t1.start();
```

```text
State = RUNNABLE
```

The thread enters the Ready Queue (Runnable Pool).

***

### Step 3: Scheduler Decision

The OS scheduler continuously checks:

```text
Ready Queue

Thread-1
Thread-2
Thread-3
Thread-4
```

and decides:

```text
CPU Core 1 ---> Thread-2
CPU Core 2 ---> Thread-4
```

Now those threads become RUNNING.

***

## How Does Scheduler Choose?

The scheduler considers:

### 1. Thread Priority

```java
t1.setPriority(10);
```

Higher priority may get preference.

But:

> Priority is only a hint, not a guarantee.

***

### 2. CPU Availability

Example:

```text
8 CPU Cores
20 Threads
```

Only 8 threads can truly run at the same instant.

The remaining 12 stay in Runnable state.

***

### 3. Time Slicing

Suppose:

```text
Thread-1 gets CPU for 10ms
```

After 10 ms:

```text
Running --> Runnable

Thread-2 --> Running
```

This switching is called **Context Switching**.

***

## Interview Answer

> **Runnable State** means the thread is ready and waiting for CPU allocation. It is present in the runnable pool or ready queue. **Running State** means the OS scheduler has selected the thread and the CPU is actively executing its `run()` method. The JVM places threads in the runnable state, but the Operating System scheduler decides when a thread gets CPU time and transitions to the running state based on factors such as CPU availability, scheduling algorithm, and thread priority.


## 3. RUNNING State

The scheduler selects the thread from the runnable pool and executes its `run()` method.


public void run() {
    System.out.println("Thread Running");
}


### Methods Used

* `run()`
* `yield()`

Example:


Thread.yield();


`yield()` suggests the scheduler give other threads a chance to execute.



## 4. WAITING / TIMED\_WAITING / BLOCKED State

### A. TIMED\_WAITING

When a thread sleeps for a specific time.


Thread.sleep(5000);


Methods:


sleep(long millis)
sleep(long millis, int nanos)


Example:


Thread.sleep(2000);


Thread pauses for 2 seconds.



### B. WAITING

Thread waits indefinitely until another thread wakes it.

Methods:


wait()
join()
park()


Example:


t1.join();


Current thread waits until `t1` finishes.



### C. BLOCKED

Occurs when a thread waits for a lock.


synchronized(obj) {
    // critical section
}


Example:


Thread 1 --> holds lock
Thread 2 --> BLOCKED waiting for lock




## 5. TERMINATED State

The thread finishes execution.


public void run() {
    System.out.println("Task completed");
}


After `run()` completes:


State = TERMINATED


The thread cannot be restarted.


t1.start();
t1.start();  // IllegalThreadStateException




# Important Thread Methods

## 1. start()

Starts a new thread.


Thread t = new Thread();
t.start();


### Interview Point

✅ Creates a new thread  
✅ Calls `run()` internally



## 2. run()

Contains business logic.


public void run() {
    System.out.println("Executing");
}


### Interview Point

If you call:


t.run();


No new thread is created.

It behaves like a normal method call.



## 3. sleep()

Pauses the current thread.


Thread.sleep(3000);


### Example


System.out.println("Start");

Thread.sleep(2000);

System.out.println("End");




## 4. join()

Waits for another thread to complete.


t1.join();


### Example


t1.start();
t1.join();

System.out.println("Main continues");


Main thread waits for `t1`.



## 5. yield()

Suggests giving CPU time to another thread.


Thread.yield();


### Example


for(int i=0;i<5;i++) {
    Thread.yield();
}


Not guaranteed to work.



## 6. isAlive()

Checks whether a thread is running.


t.start();

System.out.println(t.isAlive());


Output:


true


After completion:


false




## 7. getName() and setName()


t.setName("Worker-1");

System.out.println(t.getName());


Output:


Worker-1




## 8. currentThread()

Returns currently executing thread.


System.out.println(
    Thread.currentThread().getName()
);


Output:


main




## 9. getPriority() and setPriority()

Priority range:


1 = MIN_PRIORITY
5 = NORM_PRIORITY
10 = MAX_PRIORITY


Example:


t.setPriority(10);


### Interview Point

Priority is only a suggestion to JVM scheduler.



## 10. interrupt()

Used to interrupt a sleeping or waiting thread.


t.interrupt();


Example:


try {
    Thread.sleep(5000);
} catch (InterruptedException e) {
    System.out.println("Interrupted");
}




# Frequently Asked Interview Questions

### Q1: Difference between `start()` and `run()`?


t.start();


✅ Creates a new thread.


t.run();


✅ Executes in current thread only.



### Q2: Difference between `sleep()` and `wait()`?

| sleep()                 | wait()                              |
| ----------------------- | ----------------------------------- |
| Belongs to Thread class | Belongs to Object class             |
| Doesn't release lock    | Releases lock                       |
| Used for delay          | Used for inter-thread communication |



### Q3: Can a terminated thread be restarted?

❌ No.


t.start();
t.start(); // Exception


Throws:


IllegalThreadStateException




## Interview Summary (2-Minute Answer)

> Java Thread Life Cycle consists of **NEW → RUNNABLE → RUNNING → WAITING/BLOCKED/TIMED\_WAITING → TERMINATED** states. 
A thread is created in NEW state, moved to RUNNABLE using `start()`, executes in RUNNING state, 
may move to waiting states using methods like `sleep()`, `wait()`, and `join()`, and finally reaches TERMINATED 
when execution completes. Important thread methods include `start()`, `run()`, `sleep()`, `join()`, `yield()`, `interrupt()`,
 and `isAlive()`.
