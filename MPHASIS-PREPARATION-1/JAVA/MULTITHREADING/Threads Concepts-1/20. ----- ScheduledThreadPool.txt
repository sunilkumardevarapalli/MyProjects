If your requirement is:

* `ScheduledExecutorService`
* `ScheduledThreadPool(2)`
* Customer List
* `TriggerEmailPolicy implements Runnable`
* Store returned `ScheduledFuture<?>` objects in a `List`
* Execute every 2 minutes using `scheduleAtFixedRate()`
* Shutdown gracefully

then use the following example.

## Customer.java

```java
public class Customer {

    private int customerId;
    private String customerName;
    private String email;

    public Customer(int customerId,
                    String customerName,
                    String email) {

        this.customerId = customerId;
        this.customerName = customerName;
        this.email = email;
    }

    public int getCustomerId() {
        return customerId;
    }

    public String getCustomerName() {
        return customerName;
    }

    public String getEmail() {
        return email;
    }
}
```

***

## TriggerEmailPolicy.java

```java
public class TriggerEmailPolicy implements Runnable {

    private Customer customer;

    public TriggerEmailPolicy(Customer customer) {
        this.customer = customer;
    }

    @Override
    public void run() {

        System.out.println(
                Thread.currentThread().getName()
                + " Sending Policy Email To "
                + customer.getCustomerName()
                + " : "
                + customer.getEmail());

        try {

            Thread.sleep(3000);

        } catch (InterruptedException e) {

            System.out.println(
                    "Task Interrupted For : "
                    + customer.getCustomerName());
        }

        System.out.println(
                Thread.currentThread().getName()
                + " Email Sent To "
                + customer.getCustomerName());
    }
}
```

***

## Main.java

```java
import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.ScheduledExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.ScheduledFuture;
import java.util.concurrent.TimeUnit;

public class Main {

    public static void main(String[] args)
            throws Exception {

        List<Customer> customers =
                new ArrayList<>();

        customers.add(
                new Customer(
                        101,
                        "Sunil",
                        "sunil@gmail.com"));

        customers.add(
                new Customer(
                        102,
                        "Ravi",
                        "ravi@gmail.com"));

        customers.add(
                new Customer(
                        103,
                        "Kiran",
                        "kiran@gmail.com"));

        customers.add(
                new Customer(
                        104,
                        "Mahesh",
                        "mahesh@gmail.com"));

        customers.add(
                new Customer(
                        105,
                        "Suresh",
                        "suresh@gmail.com"));

        ScheduledExecutorService ex =
                Executors.newScheduledThreadPool(2);

        List<ScheduledFuture<?>> futureList =
                new ArrayList<>();

        for (Customer customer : customers) {

            TriggerEmailPolicy task =
                    new TriggerEmailPolicy(customer);

            ScheduledFuture<?> future =
                    ex.scheduleAtFixedRate(
                            task,
                            0,
                            2,
                            TimeUnit.MINUTES);

            futureList.add(future);
        }

        Thread.sleep(10 * 60 * 1000);

        for (ScheduledFuture<?> future
                : futureList) {

            future.cancel(true);
        }

        ex.shutdown();
    }
}
```

# Important Interview Point

If you create:

```java
ScheduledExecutorService ex =
        Executors.newScheduledThreadPool(2);
```

Only **2 worker threads** are created.

Even if:

```java
5 Customers
50 Customers
500 Customers
```

Only 2 threads execute at a time.

Remaining tasks wait in the scheduler queue.

Example:

```text
pool-1-thread-1 --> Sunil
pool-1-thread-2 --> Ravi

Waiting:
Kiran
Mahesh
Suresh
```

Once a thread is free:

```text
pool-1-thread-1 --> Kiran
```

***

# About "after 2 minutes another 10 threads will start"

This does **NOT** happen automatically.

If you create:

```java
Executors.newScheduledThreadPool(2);
```

then:

```text
Pool Size = 2
```

always.

After 2 minutes:

```text
Same 2 threads are reused
```

Java does not create 10 new threads.

Example:

```text
Time 00:00
Thread-1 -> Customer1
Thread-2 -> Customer2

Time 02:00
Same Thread-1 -> Customer1
Same Thread-2 -> Customer2

Time 04:00
Same Thread-1 -> Customer1
Same Thread-2 -> Customer2
```

The thread pool reuses the same worker threads.

### Possible Output

```text
pool-1-thread-1 Sending Policy Email To Sunil
pool-1-thread-2 Sending Policy Email To Ravi

pool-1-thread-1 Email Sent To Sunil
pool-1-thread-2 Email Sent To Ravi

pool-1-thread-1 Sending Policy Email To Kiran
pool-1-thread-2 Sending Policy Email To Mahesh

pool-1-thread-1 Email Sent To Kiran
pool-1-thread-2 Email Sent To Mahesh

pool-1-thread-1 Sending Policy Email To Suresh
pool-1-thread-1 Email Sent To Suresh
```

This is a typical real-world scheduler implementation used for:

* Policy renewal emails
* Reminder notifications
* Daily reports
* Batch processing jobs
* Cache refresh jobs
