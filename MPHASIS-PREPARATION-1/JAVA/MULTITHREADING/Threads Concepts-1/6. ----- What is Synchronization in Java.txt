## What is Synchronization in Java?

**Synchronization** is a mechanism that allows **only one thread at a time** to access a shared resource (critical section).

We use synchronization to prevent **race conditions** and ensure **data consistency** when multiple threads access the same data.

### Why Do We Need Synchronization?

Imagine a bank account balance is ₹10,000.

* Thread 1 withdraws ₹2,000
* Thread 2 withdraws ₹3,000

Without synchronization:

* Both threads may read ₹10,000 at the same time.
* Final balance could become incorrect.

With synchronization:

* One thread completes its operation first.
* The second thread gets access afterward.
* Data remains correct.

### Real-Time Definition

> Synchronization is used to control access to shared resources so that only one thread can execute a critical section at a time,
 preventing data corruption and inconsistent results.

***

## What Problems Does Synchronization Solve?

### 1. Race Condition

Multiple threads modifying the same data simultaneously.

### 2. Data Inconsistency

Shared data may become incorrect.

### 3. Thread Interference

One thread's changes affect another thread unexpectedly.

### 4. Memory Visibility Issues

Changes made by one thread may not immediately be visible to others.

# 10 Real-Time Scenarios Where Synchronization Is Used

## 1. Bank Account Transactions

Two users withdraw money from the same account.

Without synchronization:

* Balance = ₹10,000
* User A withdraws ₹5,000
* User B withdraws ₹6,000

Both may succeed incorrectly.

With synchronization:

* One transaction completes first.
* Balance is updated correctly.

***

## 2. Online Ticket Booking

100 users try to book the last seat.

Without synchronization:

* Multiple users may get the same seat.

With synchronization:

* Only one booking succeeds.

Examples:

* Movie tickets
* Flight tickets
* Train reservations

***

## 3. E-Commerce Inventory Management

Only 1 iPhone is left in stock.

Many users click "Buy Now" simultaneously.

Without synchronization:

* Stock becomes negative.

With synchronization:

* First user gets the item.
* Others receive "Out of Stock."

***

## 4. ATM Machines

Multiple ATMs access the same customer account.

Without synchronization:

* Different ATMs may show different balances.

With synchronization:

* Account balance remains consistent everywhere.

***

## 5. Employee Leave Management System

Only 2 leave slots available for a specific day.

Several employees apply simultaneously.

Without synchronization:

* More than 2 approvals may occur.

With synchronization:

* Limit is enforced correctly.

***

## 6. Payment Processing Systems

A customer accidentally clicks "Pay" twice.

Multiple payment threads start.

Without synchronization:

* Double payment may occur.

With synchronization:

* Only one payment is processed.

Examples:

* UPI
* Credit cards
* Net banking

***

## 7. Logging Frameworks

Multiple threads write logs to the same file.

Without synchronization:

* Log messages become mixed.

Example:

With synchronization:

* Logs are written properly.

***

## 8. Shared Printer in an Office

Many employees send print requests.

Without synchronization:

* Print jobs overlap.

With synchronization:

* Jobs execute one by one.

***

## 9. Hospital Appointment Booking

Only one patient can book a particular time slot.

Without synchronization:

* Same slot may be assigned to multiple patients.

With synchronization:

* Slot allocation remains accurate.

***

## 10. Database Record Updates

Two users update the same customer record.

Example:

* User A updates address.
* User B updates phone number.

Without synchronization:

* One update may overwrite the other.

With synchronization:

* Updates happen safely.

# Simple Interview Answer

> Synchronization is a mechanism in Java that restricts access to a shared resource so that only one thread can execute
 the critical section at a time. It is used to avoid race conditions, data inconsistency, and thread interference
 when multiple threads access shared data concurrently. Real-world examples include banking transactions, 
 ticket booking systems, inventory management, payment processing, ATM operations, logging systems, printer queues,
 hospital appointment booking, leave management systems, and database updates.

### Easy Rule to Remember

**If multiple threads are reading and updating the same data, synchronization is usually required.**
