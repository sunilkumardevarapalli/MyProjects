# 1. Example Using `isCancelled()` and `isDone()`

## Long Running Task

```java
import java.util.concurrent.Callable;

public class ReportTask implements Callable<String> {

    @Override
    public String call() throws Exception {

        for(int i = 1; i <= 10; i++) {

            System.out.println(
                    "Processing Step : " + i);

            Thread.sleep(1000);
        }

        return "Report Generated";
    }
}
```

## Main Class

```java
import java.util.concurrent.*;

public class Test {

    public static void main(String[] args)
            throws Exception {

        ExecutorService ex =
                Executors.newSingleThreadExecutor();

        Future<String> future =
                ex.submit(new ReportTask());

        Thread.sleep(3000);

        System.out.println(
                "Before Cancel");

        System.out.println(
                "isDone      : " + future.isDone());

        System.out.println(
                "isCancelled : " + future.isCancelled());

        future.cancel(true);

        System.out.println(
                "\nAfter Cancel");

        System.out.println(
                "isDone      : " + future.isDone());

        System.out.println(
                "isCancelled : " + future.isCancelled());

        ex.shutdown();
    }
}
```

## Possible Output

```text
Processing Step : 1
Processing Step : 2
Processing Step : 3

Before Cancel

isDone      : false
isCancelled : false

After Cancel

isDone      : true
isCancelled : true
```

***

# Understanding `isDone()`

`isDone()` returns `true` when:

* Task completed successfully
* Task failed with exception
* Task was cancelled

### Example

```java
Future<String> future =
        executor.submit(task);

future.get();

System.out.println(
        future.isDone());
```

Output:

```text
true
```

***

# Understanding `isCancelled()`

Returns:

```java
future.isCancelled();
```

### Output

```text
true
```

Only when task was cancelled.

Otherwise:

```text
false
```

***

# Real-Time Example

Suppose:

```text
Generate Insurance Report
```

Status Check:

```java
if(future.isDone()) {

    System.out.println(
            "Report Available");
}
```

Or:

```java
if(future.isCancelled()) {

    System.out.println(
            "Report Generation Cancelled");
```

# 2. What Happens if `cancel()` is Called Twice?

## Example

```java
Future<String> future =
        executor.submit(task);

boolean result1 =
        future.cancel(true);

boolean result2 =
        future.cancel(true);

System.out.println(result1);

System.out.println(result2);
```

## Output

```text
true
false
```

### Why?

First cancel:

```text
Task Cancelled Successfully
```

Second cancel:

```text
Already Cancelled
```

Nothing left to cancel.

Therefore:

```text
false
```

is returned.

***

## Example

```java
boolean first =
        future.cancel(true);

boolean second =
        future.cancel(true);

System.out.println(
        "First Cancel : " + first);

System.out.println(
        "Second Cancel : " + second);
```

Output:

```text
First Cancel : true
Second Cancel : false
```

# 3. Difference Between `cancel(true)` and `cancel(false)`

This is a favorite interview question.

***

## cancel(true)

```java
future.cancel(true);
```

Meaning:

```text
Cancel task
AND
Interrupt running thread
```

If task is:

```java
Thread.sleep()
```

or

```java
wait()
```

or

```java
join()
```

it may stop immediately with:

```text
InterruptedException
```

***

## Example

```java
public String call() throws Exception {

    Thread.sleep(10000);

    return "Completed";
}
```

Main Thread:

```java
future.cancel(true);
```

Output:

```text
Task Interrupted
```

Task stops before completion.

***

# cancel(false)

```java
future.cancel(false);
```

Meaning:

```text
Cancel task
BUT
Do not interrupt running thread
```

### Case 1

Task has not started yet

```text
Task Removed
```

Cancellation succeeds.

***

### Case 2

Task already running

```text
Running continues
```

because interruption is not allowed.

***

## Example

Suppose:

```java
Future<String> future =
        executor.submit(task);
```

Task already started.

Now:

```java
future.cancel(false);
```

Possible Result:

```text
false
```

or task may continue until completion.

***

# Visual Representation

## cancel(false)

# Complete Demonstration

```java
import java.util.concurrent.*;

public class Test {

    public static void main(String[] args)
            throws Exception {

        ExecutorService ex =
                Executors.newSingleThreadExecutor();

        Future<String> future =
                ex.submit(() -> {

                    try {

                        for(int i=1;i<=10;i++) {

                            System.out.println(
                                    "Processing " + i);

                            Thread.sleep(1000);
                        }

                    } catch (InterruptedException e) {

                        System.out.println(
                                "Task Interrupted");

                        return "Cancelled";
                    }

                    return "Completed";
                });

        Thread.sleep(3000);

        boolean cancelled =
                future.cancel(true);

        System.out.println(
                "Cancelled = "
                        + cancelled);

        System.out.println(
                "isCancelled = "
                        + future.isCancelled());

        System.out.println(
                "isDone = "
                        + future.isDone());

        ex.shutdown();
    }
}
```

### Output

```text
Processing 1
Processing 2
Processing 3

Task Interrupted

Cancelled = true
isCancelled = true
isDone = true
```

# Interview Answers

### What does `isDone()` return?

> `isDone()` returns true when the task completes normally, throws an exception, or gets cancelled.

### What does `isCancelled()` return?

> `isCancelled()` returns true only if the task was cancelled before completion.

### What happens if `cancel()` is called twice?

> The first call cancels the task and returns true. Subsequent calls return false because the task is already cancelled.

### Difference between `cancel(true)` and `cancel(false)`?

> `cancel(true)` attempts to interrupt a running task, whereas `cancel(false)` cancels the task only if it has not started yet and does not interrupt a running thread.
