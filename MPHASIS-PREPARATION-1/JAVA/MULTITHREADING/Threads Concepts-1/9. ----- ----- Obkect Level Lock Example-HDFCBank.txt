Here's a complete example based on your scenario.

### HdfcBank.java

```java
class HdfcBank {

    private int accountBalance = 10000;

    public synchronized void withdraw(int amount) {

        System.out.println(Thread.currentThread().getName()
                + " trying to withdraw: " + amount);

        if (accountBalance >= amount) {
            accountBalance = accountBalance - amount;

            System.out.println(Thread.currentThread().getName()
                    + " withdrawn: " + amount);

            System.out.println("Available Balance : "
                    + accountBalance);
        } else {
            System.out.println("Insufficient Balance");
        }
    }

    public synchronized void deposit(int amount) {

        System.out.println(Thread.currentThread().getName()
                + " trying to deposit: " + amount);

        accountBalance = accountBalance + amount;

        System.out.println(Thread.currentThread().getName()
                + " deposited: " + amount);

        System.out.println("Available Balance : "
                + accountBalance);
    }

    public void profileUpdate() {
        System.out.println(Thread.currentThread().getName()
                + " updating profile");
    }

    public void balanceCheck() {
        System.out.println(Thread.currentThread().getName()
                + " Checking Balance : "
                + accountBalance);
    }
}
```

***

### ATMThread.java

```java
class ATMThread extends Thread {

    private HdfcBank bank;

    public ATMThread(HdfcBank bank) {
        this.bank = bank;
    }

    @Override
    public void run() {

        bank.withdraw(3000);

        bank.balanceCheck();
    }
}
```

***

### GooglePayThread.java

```java
class GooglePayThread extends Thread {

    private HdfcBank bank;

    public GooglePayThread(HdfcBank bank) {
        this.bank = bank;
    }

    @Override
    public void run() {

        bank.deposit(5000);

        bank.balanceCheck();
    }
}
```

***

### Main.java

```java
public class Main {

    public static void main(String[] args) {

        HdfcBank obj1 = new HdfcBank();
        HdfcBank obj2 = new HdfcBank();

        ATMThread atm = new ATMThread(obj1);
        GooglePayThread gpay = new GooglePayThread(obj2);

        atm.setName("ATM-Thread");
        gpay.setName("GooglePay-Thread");

        atm.start();
        gpay.start();
    }
}
```

## Interview Question: What Happens Here?

You created:

```java
HdfcBank obj1 = new HdfcBank();
HdfcBank obj2 = new HdfcBank();
```

Then:

```java
ATMThread --> obj1.withdraw()
GooglePayThread --> obj2.deposit()
```

Since both methods are `synchronized`:

```java
synchronized withdraw()
synchronized deposit()
```

Locks are:

```text
obj1 lock
obj2 lock
```

Since locks are different:

```text
ATMThread  -> gets obj1 lock
GooglePay  -> gets obj2 lock
```

Both threads can execute simultaneously.

***

## If You Want Locking to Work Properly on Same Account

Use the same object:

```java
HdfcBank bank = new HdfcBank();

ATMThread atm = new ATMThread(bank);
GooglePayThread gpay = new GooglePayThread(bank);
```

Now both threads access:

```text
Same object
Same accountBalance
Same lock
```

Execution:

```text
ATMThread enters withdraw()
GooglePayThread waits

OR

GooglePayThread enters deposit()
ATMThread waits
```

Only one thread can execute a synchronized method at a time.

***

### Interview Conclusion

**Case 1: Different Objects**

```java
obj1 -> ATMThread
obj2 -> GooglePayThread
```

Result:

```text
Two Locks
Two accountBalance variables
Both threads run concurrently
```

**Case 2: Same Object**

```java
bank -> ATMThread
bank -> GooglePayThread
```

Result:

```text
One Lock
One accountBalance variable
Only one thread executes synchronized methods at a time
```

This is the classic example used to explain **object-level locking (instance-level locking)** in Java.
