## What is a Race Condition?

A **Race Condition** occurs when **two or more threads access and modify the same shared data at the same time**, 
and the final result depends on which thread finishes first.

In simple words:

> Multiple threads are "racing" to update the same resource, causing unpredictable results.

***

## Real-Time Example: Bank Account

Balance = ₹1000

Thread-1 withdraws ₹200

Thread-2 withdraws ₹300

### What should happen?

Thread-1:

* Reads balance = 1000
* Updates balance to 800

Thread-2:

* Reads latest balance = 800
* Updates balance to 500

Final balance = ₹500 ✅

***

### What may happen without synchronization?

Thread-1 reads balance = 1000

Thread-2 reads balance = 1000

Thread-1 writes 800

Thread-2 writes 700

Final balance = ₹700 ❌

Correct balance should have been ₹500.

This is a **Race Condition**.

***

## Why is it called a Race Condition?

Because threads are competing (racing) to access the same resource.

Whichever thread reaches the shared resource first affects the result.

Example:

Thread-1 → Update Database Record

Thread-2 → Update Same Database Record

The final value depends on which thread updates last.

***

## Real-Time Scenarios

### 1. Bank Withdrawals

Two users withdraw money from the same account simultaneously.

### 2. Ticket Booking

Two users try to book the last available seat.

### 3. E-Commerce Stock

Two customers buy the last iPhone at the same time.

### 4. Payment Processing

User accidentally clicks "Pay" twice.

### 5. ATM Transactions

Multiple ATMs update the same account balance simultaneously.

### 6. Employee Leave System

Several employees apply for the last available leave slot.

### 7. Shared Counter

Multiple threads increment a website visitor counter.

### 8. Online Gaming

Multiple players update the same leaderboard score.

### 9. Inventory Management

Multiple warehouse systems update stock quantities.

### 10. Database Updates

Multiple users edit the same customer record at the same time.

***

## Simple Program Example

Suppose:

count = 0

Thread-1 executes:

* Read count (0)
* Increment to 1
* Write 1

Thread-2 executes:

* Read count (0)
* Increment to 1
* Write 1

Expected result:

count = 2

Actual result:

count = 1

One update is lost.

This is called a **Lost Update Problem**, which is a type of Race Condition.

***

## How Do We Prevent Race Conditions?

### 1. Synchronization

Only one thread can access the critical section at a time.

### 2. Locks

Examples:

* ReentrantLock
* ReadWriteLock

### 3. Atomic Classes

Examples:

* AtomicInteger
* AtomicLong

### 4. Concurrent Collections

Examples:

* ConcurrentHashMap
* CopyOnWriteArrayList

***

## Interview Answer (30 Seconds)

> A Race Condition occurs when multiple threads simultaneously access and modify the same shared resource, 
and the final result depends on the order in which the threads execute. 
It can lead to inconsistent or incorrect data. Race conditions are typically prevented using synchronization, locks, or atomic classes.
