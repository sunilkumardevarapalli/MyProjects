## Disadvantages of Multithreading (Threads)

Although multithreading improves performance and responsiveness, it also introduces several challenges.

### 1. Race Condition

Occurs when multiple threads access and modify shared data simultaneously.

```java
count++;
```

Suppose:

* Thread 1 reads `count = 5`
* Thread 2 reads `count = 5`
* Both increment and write back `6`

Expected:

```text
7
```

Actual:

```text
6
```

### Real-Time Scenario

Bank account balance updates from multiple transactions at the same time.

***

### 2. Deadlock

Two or more threads wait forever for each other's resources.

Example:

```text
Thread 1 holds Lock A and waits for Lock B
Thread 2 holds Lock B and waits for Lock A
```

Result:

```text
Application hangs forever
```

### Real-Time Scenario

Payment service and inventory service locking shared resources.

***

### 3. Context Switching Overhead

The CPU frequently switches between threads.

```text
Thread 1 → Thread 2 → Thread 3 → Thread 1
```

Each switch requires saving and restoring thread state.

### Impact

* Increased CPU overhead
* Reduced performance if too many threads exist

### Real-Time Scenario

Creating thousands of threads in a web application.

***

### 4. Complex Debugging

Multithreaded bugs are difficult to reproduce.

Example:

```java
Sometimes works
Sometimes fails
```

The issue may occur only when threads execute in a specific order.

### Real-Time Scenario

Intermittent production issues that cannot be reproduced in development.

***

### 5. Synchronization Overhead

To avoid race conditions, we use:

```java
synchronized
Lock
Semaphore
```

These mechanisms introduce delays.

### Impact

```text
More locking
↓
Less concurrency
↓
Lower performance
```

***

### 6. Memory Consumption

Each thread has its own stack memory.

Example:

```text
1 Thread  -> ~1 MB Stack
1000 Threads -> ~1 GB Stack
```

(Actual size depends on JVM settings.)

### Real-Time Scenario

Microservice creating too many threads resulting in OutOfMemoryError.

***

### 7. Starvation

A thread never gets CPU time because other threads continuously consume resources.

### Real-Time Scenario

```text
High-priority threads keep running
Low-priority thread keeps waiting
```

***

### 8. Livelock

Threads are active but make no progress.

Example:

```text
Thread 1: You go first!
Thread 2: No, you go first!
```

Both keep responding to each other and never complete the work.

***

### 9. Resource Contention

Multiple threads compete for:

* Database connections
* Files
* Network sockets
* Shared objects

### Real-Time Scenario

500 users trying to update the same record simultaneously.

***

### 10. Difficult Testing

Single-threaded applications are predictable.

Multithreaded applications are not.

```text
Run 1 -> Success
Run 2 -> Failure
Run 3 -> Success
```

The execution order may differ every time.

***

## Interview Answer (2 Minutes)

> The major disadvantages of multithreading are race conditions, deadlocks, synchronization overhead, context switching costs, increased memory consumption, starvation, livelock, and complex debugging. Although multithreading improves responsiveness and performance, improper thread management can lead to data inconsistency, application hangs, and scalability issues. Therefore, Java applications often use `ExecutorService`, thread pools, and concurrent collections to manage threads efficiently.

### Interview Tip

If asked **"What's the biggest disadvantage of multithreading?"**, answer:

> **Concurrency issues such as race conditions and deadlocks are the biggest challenges because they can cause incorrect results or completely freeze an application and are often difficult to debug.**
