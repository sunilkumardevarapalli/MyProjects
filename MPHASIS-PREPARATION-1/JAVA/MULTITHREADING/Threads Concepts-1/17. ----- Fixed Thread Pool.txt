This is a very common **real-time bulk email processing** interview scenario.

### Customer.java

```java
public class Customer {

    private int customerId;
    private String customerName;
    private String email;

    public Customer(int customerId,
                    String customerName,
                    String email) {

        this.customerId = customerId;
        this.customerName = customerName;
        this.email = email;
    }

    public int getCustomerId() {
        return customerId;
    }

    public String getCustomerName() {
        return customerName;
    }

    public String getEmail() {
        return email;
    }
}
```

### TriggerEmailPolicy.java

```java
public class TriggerEmailPolicy implements Runnable {

    private Customer customer;

    public TriggerEmailPolicy(Customer customer) {
        this.customer = customer;
    }

    @Override
    public void run() {

        triggerEmail(customer);
    }

    private void triggerEmail(Customer customer) {

        System.out.println(
                Thread.currentThread().getName()
                + " Sending Policy Email To : "
                + customer.getCustomerName()
                + " -> "
                + customer.getEmail());

        try {
            Thread.sleep(3000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        System.out.println(
                Thread.currentThread().getName()
                + " Email Sent To : "
                + customer.getCustomerName());
    }
}
```

### Main.java

```java
import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class Main {

    public static void main(String[] args) {

        List<Customer> customers = new ArrayList<>();

        customers.add(
                new Customer(
                        101,
                        "Sunil",
                        "sunil@gmail.com"));

        customers.add(
                new Customer(
                        102,
                        "Ravi",
                        "ravi@gmail.com"));

        customers.add(
                new Customer(
                        103,
                        "Kiran",
                        "kiran@gmail.com"));

        customers.add(
                new Customer(
                        104,
                        "Mahesh",
                        "mahesh@gmail.com"));

        customers.add(
                new Customer(
                        105,
                        "Suresh",
                        "suresh@gmail.com"));

        ExecutorService ex =
                Executors.newFixedThreadPool(3);

        for (Customer customer : customers) {

            TriggerEmailPolicy t =
                    new TriggerEmailPolicy(customer);

            ex.execute(t);
        }

        ex.shutdown();
    }
}
```



### Interview Explanation

```java
ExecutorService ex =
        Executors.newFixedThreadPool(3);
```

Creates **3 worker threads**.

```java
ex.execute(t);
```

Submits a Runnable task to the thread pool.

```java
ex.shutdown();
```

Stops accepting new tasks, but allows already submitted tasks to complete.

### Flow

```text
Customer List
      ↓
for loop
      ↓
Create Runnable Task
      ↓
execute(task)
      ↓
Thread Pool Queue
      ↓
Available Thread Picks Task
      ↓
triggerEmail()
      ↓
Email Sent
```

This is exactly how bulk email, SMS, notification, and policy-renewal jobs are implemented in many enterprise applications.
