## Scenario: FixedThreadPool with 2 Threads

Let's modify your example to use:

```java
ExecutorService ex = Executors.newFixedThreadPool(2);
```

and add delay to simulate a real email sending operation.

***

## TriggerEmailPolicy.java

```java
public class TriggerEmailPolicy implements Runnable {

    private Customer customer;

    public TriggerEmailPolicy(Customer customer) {
        this.customer = customer;
    }

    @Override
    public void run() {

        try {

            System.out.println(
                    Thread.currentThread().getName()
                    + " Started Email For "
                    + customer.getCustomerName());

            // Simulate email processing
            Thread.sleep(5000);

            System.out.println(
                    Thread.currentThread().getName()
                    + " Email Sent To "
                    + customer.getCustomerName());

        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}
```

***

## Main.java

```java
import java.util.Arrays;
import java.util.List;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class Main {

    public static void main(String[] args) {

        List<Customer> customers = Arrays.asList(

                new Customer(101, "Sunil", "sunil@gmail.com"),
                new Customer(102, "Ravi", "ravi@gmail.com"),
                new Customer(103, "Kiran", "kiran@gmail.com"),
                new Customer(104, "Mahesh", "mahesh@gmail.com"),
                new Customer(105, "Suresh", "suresh@gmail.com")
        );

        ExecutorService ex =
                Executors.newFixedThreadPool(2);

        for(Customer customer : customers) {

            TriggerEmailPolicy task =
                    new TriggerEmailPolicy(customer);

            ex.execute(task);
        }

        ex.shutdown();
    }
}
```


# Why Use FixedThreadPool(2)?

Without Thread Pool:

```text
5 Customers
↓
5 Threads Created
```

Problems:

* More memory
* Context switching overhead
* Thread creation cost

***

With FixedThreadPool(2):

```text
5 Tasks
↓
Only 2 Threads Created
↓
Threads Reused
↓
Better Performance
```

***

## Important Interview Question

### How many threads are created?

```java
Executors.newFixedThreadPool(2);
```

Answer:

```text
Only 2 worker threads
```

regardless of:

```text
5 Tasks
50 Tasks
500 Tasks
```

Extra tasks go into the queue and wait for an available thread.

***

## Interview Answer

> `FixedThreadPool(2)` creates exactly two worker threads.
 When five customer email tasks are submitted,
 the first two tasks are executed immediately and the remaining tasks are placed in an internal queue. 
 As soon as a worker thread finishes its current task, it picks the next task from the queue.
 This process continues until all tasks are completed. The threads are reused, reducing thread creation overhead and 
 improving application performance.
