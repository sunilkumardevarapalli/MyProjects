Java is called **platform independent** because of the **Java Virtual Machine (JVM)**.

### How it works

1. **Write Java code**

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
```

2. **Compile the code**

```bash
javac Hello.java
```

This produces **bytecode** (`Hello.class`).

3. **Run the bytecode on any platform**

```bash
java Hello
```

The **JVM** on Windows, Linux, or macOS interprets or JIT-compiles the same bytecode into native machine code for that platform.

### Architecture

```
Java Source Code (.java)
          |
       javac
          |
       Bytecode (.class)
          |
   -----------------------
   |         |          |
 Windows JVM Linux JVM macOS JVM
   |         |          |
Machine Code for each OS
```

### Why Java is platform independent?

* **Compiler generates bytecode, not machine code.**
* **Bytecode is OS-independent.**
* **Each platform has its own JVM implementation.**
* Same `.class` file can run anywhere a JVM is available.

This is known as:

> **WORA (Write Once, Run Anywhere)**

### Interview Answer (10+ Years Experience)

> Java is platform independent because the Java compiler converts source code into platform-neutral bytecode. The bytecode is executed by the JVM, which is platform specific. Since every operating system provides its own JVM implementation, the same bytecode can run unchanged across different platforms, achieving the "Write Once, Run Anywhere" capability.

### Platform Independent vs Platform Dependent

| Language   | Compilation Output     | Platform Independent |
| ---------- | ---------------------- | -------------------- |
| Java       | Bytecode (.class)      | ✅ Yes                |
| C          | Machine code           | ❌ No                 |
| C++        | Machine code           | ❌ No                 |
| Python     | Bytecode + Interpreter | ✅ Yes                |
| JavaScript | Engine-based execution | ✅ Yes                |

**Important:** Java is **platform independent**, but the **JVM itself is platform dependent** (there are separate JVM implementations for Windows, Linux, macOS, etc.).
